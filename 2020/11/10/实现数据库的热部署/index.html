【笔记】实现数据库的热部署

发表于 2020-11-10 更新于 2021-03-21 分类于 Cloud Computing 阅读次数：

前言

实现Mysql（MariaDB）数据库的主从搭建，从而实现热部署

配置主机和从机

  • 准备两台数据库服务器

主机配置

  • 编辑主机my.cnf配置文件
  • 修改主机my.cnf配置文件，在[mysqld]区域追加
  • 重启mysql服务
  • 登陆mysql，查看主机的MASTER状态，并记录，将其配置到从机

从机配置

  • 编辑从机my.cnf配置文件
  • 修改从机my.cnf配置文件，在[mysqld]区域追加
  • 重启mysql服务
  • 登陆数据库，实现数据库的主从挂载

MASTER_HOST：主机ip
MASTER_PORT：主机端口号
MASTER_USER：主机用户名
MASTER_PASSWORD：主机密码
MASTER_LOG_FILE：主机FILE，从主机的状态中获取
MASTER_LOG_POS：主机POS，从主机的状态中获取

  • 启动数据库的主从服务
  • 检查数据库的启动状态，查看Slave_IO_Running和Slave_SQL_Running的值是否为YES

完成

后记

实现双机热备

  • 为了实现数据库的高可用，可以实现双机热备（双主模式）
  • 将当前集群的主机配置为从机，当前集群的从机配置为主机，在进行一次热部署配置
  • 保证两台数据库相互是另一台数据库的主机，也是另一台数据库的从机。
