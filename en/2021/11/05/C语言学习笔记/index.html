Introduction

C is a general-purpose programming language widely used in the development of system software and application software. It was designed and developed by Dennis Ritchie and Ken Thompson at Bell Labs between 1969 and 1973, based on the B language, for the purpose of porting and developing the UNIX operating system. C language is favored in program design for its characteristics of efficiency, flexibility, rich functionality, strong expressiveness, and high portability, becoming the most widely used programming language in the past 25 years. (Wikipedia)

  • Include a header file

main Function

  • The main function is the entry point of a C program.
  • A C program must have one and only one main function.

Comments

Line Comment

  • This type of comment is inherited from C++ and can now be used in C language.

Block Comment

  • This type of comment is the standard way of commenting in C language.

Variable Naming Rules

  • Allowed characters include letters, numbers, and underscores.
  • The first character cannot be a number.
  • Cannot be a C language built-in keyword.

print Function

  • Outputs a string to a standard output device.

<str>: Any type of data

Compilation Errors

Warning

  • A warning that does not affect compilation.

Error

  • An error that prevents compilation.

return Keyword

  • Return value.
  • If the return type is void, no return value is needed.
  • Execution stops immediately at the return statement, and any code after return has no effect.

return in the main Function

  • Calling the return keyword in the main function immediately terminates the program.
  • return 0 indicates successful execution.
  • return -1 indicates program failure.

System Function

  • Invoking system allows running another program within a C program.

<command>: Command specific to the current operating system.

Cross-Platform Considerations

  • In different platforms, C language can guarantee consistent syntax in source code, but not necessarily the same execution results.
  • To address cross-platform issues, POSIX standards can be used for writing code.
    • Windows has poor support for POSIX standards.
  • When possible, use functions that comply with POSIX standards.

Operating System Structure

User Mode

  • Regular C programs can only access their own memory.

Kernel Mode

  • Only low-level C programs can access memory related to the system kernel.

CPU

Difference Between 32-bit and 64-bit Systems

  • 32-bit CPU
    • 32 registers can hold 32 binary bits.
    • 32-bit CPU has a 32-bit bus.
    • Maximum memory capacity managed by 32-bit is 4GB.
      • Therefore, installing a 64-bit OS on a system with less than or equal to 4GB of memory doesn’t necessarily offer faster performance compared to a 32-bit system.
  • By installing a 64-bit OS on a 64-bit CPU, the system becomes 64-bit.
  • 64-bit CPU
    • 64 registers can hold 64 binary bits.
    • 64-bit CPU has a 64-bit bus.
    • The maximum memory capacity managed by 64-bit is at least greater than 4GB.
  • By installing a 32-bit OS on a 64-bit CPU, the system is 32-bit.
  • Quasi-64-bit CPU
    • 64 registers with a 32-bit bus.
    • Operations can achieve 64-bit effectiveness, but data exchange externally can only be done at 32-bit speed.

Unsigned and Signed Numbers

Unsigned Numbers

  • Can only represent positive numbers.
  • Range: 0 to 255

Signed Numbers

  • The first bit is a sign bit, allowing for representation of negative numbers.
  • Range: -125 to 124
Signed Magnitude
  • The first bit is the sign bit, and the remaining bits represent the magnitude.
  • Sign bit for positive numbers is 0, for negative numbers is 1
Ones’ Complement
  • For positive numbers, ones’ complement is the same as signed magnitude.
  • For negative numbers, the first bit is the sign bit, with the other bits being the one’s complement of the magnitude.
    • Sign bit for positive numbers is 0, for negative numbers is 1
Two’s Complement
  • For positive numbers, two’s complement is the same as ones’ complement.
  • For negative numbers, the first bit is the sign bit, and the rest of the bits are the one’s complement plus 1.
    • Sign bit for positive numbers is 0, for negative numbers is 1

sizeof Keyword

  • Determines the size of a variable in memory.
  • Returns an unsigned number.

Endianness

  • Little-Endian (reverse): High memory stores integer high-order bits, while low memory stores integer low-order bits.
    • x86 and ARM CPUs typically use Little-Endian storage.
  • Big-Endian (normal): Low memory stores integer high-order bits, while high memory stores integer low-order bits.
    • Unix CPUs typically use Big-Endian storage.

Type Qualifiers

const Keyword

  • Prevents variables from being modified; can be understood as a constant.

volatile Keyword

  • When a variable is changed frequently, the compiler automatically optimizes the code. This optimization can have its own drawbacks. To prevent the compiler from optimizing, the volatile keyword can be used.

register Keyword

  • Directly places variables in registers for faster access.
