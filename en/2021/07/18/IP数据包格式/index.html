【英文】IP数据包格式

Posted on 2021-07-18 Edited on 2024-03-28 In 计算机网络学习指北 Views:

Preface

IP Packet Format

IP Packet

  • IP Packet (Layer 3 Data) Header

Version

  • Indicates IPv4 or IPv6

Header Length

  • Indicates the number of bytes in the entire IP header
  • The length range of the header is 20-60 bytes, depending on the preference

Priority and Service Type

  • Usually used to limit network speed
  • Priority is 3 bits, service type is 4 bits, and the remaining 1 bit is reserved

Total Length

  • Indicates the total length of the entire IP packet (including Layer 3 header, Layer 4 header, and Layer 5 data), in bytes

Fragment Offset

  • If the data in an IP packet is large (including the IP header), exceeding 1500 bytes, the data will be fragmented. Each fragment can carry a maximum of 1480 bytes of data (including the Layer 4 header and Layer 5 data). When the receiving network card receives multiple fragmented data, it needs to reassemble the data according to the fragment offset to control the order of the data.
  • The fragment offset for the 1st fragment is 14800=0 bytes, the fragment offset for the 2nd fragment is 14001=1480 bytes, and so on.

Identifier

  • If the data in multiple IP packets is large and fragmentation is involved, it is easy to confuse the order of the fragments if only the fragment offset is used. Identifier can be used to distinguish them.
  • The identifier is a randomly generated ID.

Flags

  • The first bit is always 0 (reserved)
  • The second bit
    0 indicates fragmentation has occurred
    1 indicates no fragmentation
  • The third bit
    1 indicates it is not the last fragment
    0 indicates it is the last fragment

TTL (Time to Live)

  • Time left before the packet is discarded

  • Upon passing through a router, the TTL value decreases by 1. When it reaches 1, it is no longer decreased and the packet is discarded.

  • Initial value:
    For Microsoft series operating systems, it is usually greater than 100.
    For Unix series operating systems, it is usually less than 100.
    For other network devices, it is usually equal to 255.

  • Purpose: To prevent IP packets from circulating indefinitely on the Internet

Protocol Number

  • Identifies the upper layer protocol

If the upper layer is TCP protocol, the protocol number is 6.
If the upper layer is UDP, the protocol number is 17.
If the upper layer is ICMP, the protocol number is 1.

Header Checksum

  • Verifies if the IP header has been tampered with

Source Address

  • Source IP address

Destination Address

  • Destination IP address
