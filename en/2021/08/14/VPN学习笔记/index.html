【英文】VPN学习笔记

Posted on 2021-08-14 Edited on 2024-03-28 In 计算机网络学习指北 Views:

Introduction

VPN (Virtual Private Network) is a virtual private network.
VPN can securely transmit data over the Internet and ensures the security of data through PKI technology.

Types of VPN

  • Remote Access VPN

Generally used for secure connections from individuals to enterprise internal network.

  • Point-to-Point VPN

Generally used for secure connections between enterprises.

VPN Tunneling Techniques

Transport Mode

  • Encrypts only the upper layer and does not encrypt the private IP packet header, resulting in faster speed compared to tunnel mode.

Tunnel Mode (Default Value)

  • Encrypts the entire private IP packet, including the IP packet header, providing higher security but slower speed compared to transport mode.

Configuration Commands for IPsec VPN (Point-to-Point VPN)

  • Generally divided into 2 phases.

Management Connection

  • Both communicating devices encrypt the symmetric key used by the symmetric algorithm with asymmetric encryption algorithm.

Enter the Transport-Level Configuration Mode

  • In the global configuration mode, enter the transport-level policy configuration mode.

<num>: Transport set, policy set. Defines the table number of the policy set.

Configure Encryption Algorithm (Agreed by both parties)

  • In the transport-level configuration mode.

<encryption>: Defines the encryption algorithm, des/3des/aes. Usually aes is used.

Configure Integrity Algorithm (Agreed by both parties)

  • In the transport-level configuration mode.

<whole>: Defines the integrity algorithm, md5/sha. Usually sha is used.

Configure the Length of Asymmetric Encryption Algorithm (Agreed by both parties)

  • In the transport-level configuration mode.

<length>: Defines the length of the asymmetric encryption algorithm, 1/2/5. Usually 2 is used, indicating 1024 bytes.

Configure the Use of Pre-Shared Keys (Agreed by both parties)

  • In the transport-level configuration mode.
  • Authentication method does not use asymmetric encryption algorithm but uses pre-shared keys to ensure that the hash value of the authentication is not tampered with.

Configure the Expiration Time (Optional)

  • In the transport-level configuration mode.

<second>: Time in seconds, default value is 86400 seconds.

Perform Negotiation

  • Return to the global configuration mode to perform configuration.

<pwd>: Pre-shared key. Customizes a password with a length of more than 6 digits and as complex as possible (agreed by both parties).
<ip>: Public IP address of the other party.

Data Connection

  • Encrypts the actual private network data to be transmitted using symmetric encryption algorithm.

Define VPN Trigger Traffic

  • In the global configuration mode.
  • Configure which network segments can go through the tunnel.

100: Table name.
<ip_1>: Source (private network) IP network segment.
<mask_1>: Source reverse subnet mask.
<ip_2>: Destination (private network) IP network segment.
<mask_2>: Destination reverse subnet mask.

Define Encryption and Authentication Method (Agreed by both parties)

  • In the global configuration mode.

<name>: Name of the transport mode, custom name.
<support>: Support options.

esp: Supports encryption and authentication.
ah: Supports authentication only. Less commonly used in China.

<encryption>: Defines the encryption algorithm, des/3des/aes.
<whole>: Defines the integrity algorithm, md5/sha.

Application Phase

Create MAP Mapping Table

  • In the global configuration mode, enter the MAP table configuration mode.
  • Map Phase 1 and Phase 2 and apply them to the port.
  • At this point, the IPsec VPN takes effect.

<mapname>: MAP table name.
1: MAP table identifier.

Match VPN Traffic

  • In the MAP table configuration mode.

100: Table name defining the VPN trigger traffic.

Match Encryption and Authentication Method

  • In the MAP table configuration mode.

<name>: Table name of the encryption and authentication method.

Configure the Public IP of the Other Party

  • In the MAP table configuration mode.

<ip>: Public IP of the other party.

Apply the MAP table to the WAN port

  • In the port configuration mode.

<mapname>: MAP table name.

Only one MAP table can be applied to one interface (Attention)

  • In order to solve the problem of multiple VPN tunnels on one interface, when setting the MAP table name, set the same table name, and when setting the table identifier, set different table identifiers. Use the table identifier to distinguish different MAP tables, thereby solving the problem of multiple VPN tunnels on one interface.
