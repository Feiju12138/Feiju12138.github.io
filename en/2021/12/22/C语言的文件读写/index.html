Open File

  • Create a pointer variable for file operation, the data type of the variable is FILE

  • If the file is successfully opened, a valid memory address is returned; if the opening fails, the return value is NULL

  • On Windows

    • If handling text files, do not add the b parameter to let the program automatically handle line breaks \r\n
    • If handling binary files (not just writing), add the b parameter to prevent the program from automatically handling line breaks \r\n and do not add unnecessary \r when manually writing data

  • On Linux, the b parameter has no effect, so there is no need to add the b parameter

<src>: file path
<mode>: open mode

Parameter Read/Write option Is binary Error when file does not exist File pointer position Note
r read only error File must be readable
r+ read/write error File must be readable
rb+ read/write binary error Only effective on Windows, used to read \r in line breaks \r\n on Windows
rw+ read/write error Only effective on Windows, used to read \r in line breaks \r\n on Windows
w write only create file beginning of file
wb write only binary create file beginning of file
w+ read/write create file beginning of file
a write only create file end of file EOF is preserved
a+ read/write create file end of file EOF is not preserved

Manually Manipulate File Pointers

Move File Pointer

  • By default, the file pointer starts from the beginning and automatically moves forward each time
  • If you want to manually manipulate the file pointer, you can use the fseek function

p: file pointer variable
<num>: the number of units the pointer moves from the starting position of the pointer
<location>: the starting position the pointer moves from

SEEK_SET: set the starting position of the pointer movement to the beginning of the file
SEEK_CUR: set the starting position of the pointer movement to the current position
SEEK_END: set the starting position of the pointer movement to the end of the file

  • If successful, returns 0
  • If failed, returns a non-zero value, and the pointer position remains unchanged
  • If the maximum position is exceeded, it still returns 0, and when moving forward again (not moving into the file), it still returns 0

Write Data to File Directly

  • When writing data to a file in C, it is written to the buffer by default, aiming to reduce the number of disk read and write operations, extend the lifespan of the disk, and improve efficiency
  • When the data written exceeds the size of the buffer, or when the fclose function is called, the C program will immediately write the data to the disk
  • If you want to write data directly to the disk without using the buffer, you can use the fflush function
    • The fflush function can immediately write the data in the buffer to the disk

p: file pointer variable
