Introduction

Notes on the study of tree data structures

The Concept of Trees

Degree of a Node

  • The number of subtrees that a node has is called its degree.

Leaf Node (Terminal Node)

  • A node with a degree of 0 is called a leaf node.

Branch Node

  • A node with a degree not equal to 0 is called a branch node.

Level of a Node

  • Starting from the root node, the level of the root node is 1, the immediate successors of the root node are at level 2, and so on.

Level Order Number of a Node

  • Arrange the nodes in the tree in a linear sequence from top to bottom and from left to right in the same level, and assign them natural numbers.

Degree of a Tree

  • The maximum degree of all nodes in a tree.

Height (Depth) of a Tree

  • The maximum level of nodes in a tree.

Forest

  • A collection of m (m>=0) mutually disjoint trees. If the root node of a non-empty tree is deleted, the tree becomes a forest. If a unified root node is added to the forest, the forest becomes a single tree.

Child Node

  • The immediate successor nodes of a node are called its child nodes.

Parent Node

  • The immediate predecessor of a node is called its parent node.

Sibling Node

  • The child nodes of the same parent node are called sibling nodes.

Binary Tree

  • A binary tree is a tree with a degree not exceeding 2.

Full Binary Tree

  • If every level of the binary tree has the maximum number of nodes, the binary tree is called a full binary tree.
  • The number of nodes in each level is 2^(k-1).

Complete Binary Tree

  • The leaf nodes can only appear on the last and second-to-last levels, and the nodes on the last level are concentrated on the leftmost positions of that level in the binary tree.

2-3 Tree

Definition

2- Node

  • A node that contains one key and two links. The left link points to a child node that is smaller than the parent node, and the right link points to a child node that is larger than the parent node.

3- Node

  • A node that contains two keys and three links. The left link points to a child node that is smaller than the smaller key of the parent node, the middle link points to a child node that is larger than the smaller key but smaller than the larger key of the parent node, and the right link points to a child node that is larger than the larger key of the parent node.

Properties of 2-3 Trees

  • The path length from any empty link to the root node is equal.

  • When a 4-node is transformed into a 3-node, the height of the tree remains the same. Only when the root node is a temporary 4-node and is split, the height of the tree increases by 1.

  • The biggest difference between a 2-3 tree and a regular binary tree is that a regular binary tree grows from top to bottom, while a 2-3 tree grows from bottom to top.

Red-Black Tree

  • Red-black tree is mainly an encoding of 2-3 trees. The basic idea behind red-black trees is to use a standard binary tree (composed entirely of 2-nodes) and colors to represent 2-3 trees.

Red Link

  • Connects two 2-nodes to form a 3-node.

Black Link

  • Regular link in a 2-3 tree.

Definition of Red-Black Tree

  • All links are left links.

  • No node has two red links connected to it.

  • A red-black tree is perfectly balanced in black, which means the number of black links on any path from an empty link to the root node is the same (the parent and child nodes connected by red links are considered as nodes on a black link).

Balancing Operations

Left Rotation

  • When a node has a black left child and a red right child, a left rotation is needed to balance the red-black tree.

Right Rotation

  • When a node has two red children and the left child’s left child is also red, a right rotation is needed to balance the red-black tree.

Color Flip

  • When a node has two red children, in order to balance the red-black tree, we can change the color of the parent node’s link to red and change the color of all links from the parent node to the children nodes to black.

B-Tree

  • B-tree is essentially an extension of 2-3 trees. B-trees allow multiple key-value pairs in a single node.

Properties of B-Trees

  • Each node can have at most m-1 keys, and they are sorted in ascending order.
    m: total number of key-value pairs in the current node

  • Each node can have at most m children.

  • The root node must have at least two children.

  • In practical applications, the order of B-trees is usually large (usually greater than 100), so even with a large amount of data, the height of B-trees remains relatively small.
    Order: the maximum number of children for each node

B+ Tree

  • B+ tree is a variant of B-tree.

Differences Between B+ Tree and B-Tree

  • Non-leaf nodes are used only for indexing, which means they only store keys and do not store values.

  • All leaf nodes of the tree form an ordered linked list, and all data can be traversed in the order of key sorting.

Disjoint-Set

Structure of Disjoint-Set

  • Each element is uniquely associated with a node.

  • Multiple elements in a group are in the same tree.

  • There is no connection between the trees of different groups.

  • Elements in a tree do not have a rigid requirement on the parent-child relationship.
