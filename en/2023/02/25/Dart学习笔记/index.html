Constants

const

  • Constants defined with const are determined at compile time and will never change
  • If two constants with the same value are const-qualified, they will share the same memory space

final

  • Constants defined with final can be declared first and then assigned, and they cannot be changed once assigned
  • final not only has the characteristics of const, but final is lazily initialized

Defining Constant Constructors

  • Constant constructors must be qualified with const
  • Constant constructors must be used for classes where all member attributes are qualified with final
  • If you want to share memory space, you need to create objects using a constant constructor
