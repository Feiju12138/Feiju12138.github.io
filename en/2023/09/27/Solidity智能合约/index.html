Introduction

Notes on Solidity Smart Contract

Development Tools

  • https://plugins.jetbrains.com/plugin/9475-solidity

Writing Smart Contracts

  • Create a smart contract, define state variables, define functions
  • Solidity code ends with .sol and needs to be deployed and executed on the blockchain
  • The smart contract name must be the same as the file name
  • The first line of the file // SPDX-License-Identifier: GPL-3.0 defines the open-source license
  • Define the Solidity compiler version with pragma solidity ^0.8.0;

Paying Gas Costs When Deploying

  • If it is a development environment:
    • If the configuration file does not specify a payment account, the gas costs will be paid by the first account on the test chain
    • If the configuration file specifies a payment account, the gas costs will be paid by that account
  • If it is a production environment, the gas costs will be paid by the caller of the smart contract

Variable Scopes

  • State variables: Variables inside the smart contract, outside the functions, are state variables. All functions within the contract can access them. They are stored on the blockchain and consume a lot of gas
  • Local variables: Variables inside functions are local variables. Local variables are only valid during the execution of the function and become invalid after the function execution ends. The data of local variables is stored in memory, not on the blockchain, and consumes less gas
  • Global variables: Variables reserved by Solidity
    • msg.sender: Public key (address) of the connecting account
    • block.number: Block number

Visibility Modifiers

  • Visibility modifiers can be applied to functions or state variables
Identifier Accessible by External Contracts Accessible by Internal Contracts Accessible by Child Contracts
external Yes No No
public Yes Yes Yes
internal No Yes Yes
private No Yes No

Applying to State Variables

  • If a state variable is set to public, a Getter method will be automatically added and assigned to that state variable
  • Call the state variable as a function to get its value

Applying to Functions

Data Location

  • Primitive types do not need to specify the data location

Data Locations

storage: Default value, on the blockchain, state variables exist in storage
memory: Not on the blockchain, function parameters and temporary variables can be stored in memory, variables can be modified, consumes less gas
calldata: Not on the blockchain, function parameters and temporary variables can be stored in calldata, variables cannot be modified, consumes less gas

Function Types

Function Types

No identifier: Default value, on the blockchain, can read and write state variables in the function

Function Types

view: Not on the blockchain, can only read state variables in the function
pure: Not on the blockchain, cannot read or write state variables in the function
