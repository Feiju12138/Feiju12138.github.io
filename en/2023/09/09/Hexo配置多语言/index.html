【英文】Hexo配置多语言

Posted on 2023-09-09 Edited on 2024-03-28 Views:

Preface

Hexo 6.3 configures multi-language support (i18n) under the NexT 8.18 theme.

Install the plugin to render the homepage

  • When only the hexo-generator-index plugin is installed, the language option in _config.yml cannot be configured with multiple values, so it is not possible to configure multiple languages for the site at the same time.

Install the plugin that allows multiple languages

  • After installing the hexo-generator-i18n plugin, the language option in _config.yml allows multiple values to be configured, so multiple languages can be configured for the site at the same time.

Configure multiple languages

Configure translated pages (optional)

  • After installing the hexo-generator-i18n plugin, you can configure which paths or types of pages will be automatically translated. If not configured, only the homepage will be translated.

i18n.type: Specifies which types of files will be automatically translated.

post: All post pages.
page: All non-post pages.

i18n.generator: Specifies which paths of files will be automatically translated.

index: Homepage.
archive: Archive page.
category: Category page.
tag: Tag page.

Configure translations (optional)

  • Even if not configured, NexT provides default translations for multiple languages.

Link

Install the plugin for single-language homepage

  • Before installing the hexo-generator-index-i18n plugin, although the site can display articles in two languages at the same time, it will not separate the articles in multiple languages. Instead, all articles will be displayed on the homepage.

Configure the single-language homepage

  • Make the homepage display only one language, thus separating articles in multiple languages into different language homepages.

Redefine the resource access path

  • After configuring, the previously accessed resource request URL path changes from http://localhost:4000/year/month/day/post-title to http://localhost:4000/language/year/month/day/post-title.

Reorganize old articles by language

  • Add the lang tag to all articles.

lang: Configures the language of the current article. Since the permalink configuration has been modified, the resource path is automatically divided based on the language of the current article when accessing the resource.

Batch add language tags script

Store articles in separate directories based on language (optional)

  • After configuring the lang tag, although it is possible to separate resource request URL paths, the files stored locally are still all in the hexo/source/_post directory, without being classified according to language
  • Manually put the files of articles in different languages into corresponding language directories
    • This operation does not change the resource request URL path, but only facilitates management. The actual language in the resource request URL path is based on the lang tag in the article

Configure default lang tag for new articles

  • From now on, new articles created with hexo new will be automatically added with the default lang tag
    • Since the default is set to zh-CN, the created article is initially a Chinese article. To change it to an English article, simply change zh-CN to en

Create articles and store them in corresponding language directories (optional)

  • After configuring the default lang tag, although it is possible to separate resource request URL paths, the newly created articles are not automatically moved to the corresponding directories. When creating articles with hexo new, they are still placed in the hexo/source/_post directory

  • Automatically store newly created files in the corresponding language directories by modifying the configuration

  • After configuring, when creating new articles using hexo new, they will be automatically stored in the hexo/source/_post/undefined directory because no language directory is specified. Only when creating articles with the --lang parameter specifying the article language can they be placed in the corresponding directory

  • If switching back to Chinese, the URL does not have a language prefix by default, and it will jump to an invalid page
  • Manually force adding /zh-CN as the URL prefix
