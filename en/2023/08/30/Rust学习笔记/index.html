Managing projects with Cargo

Checking Cargo version

  • Cargo is installed together with Rust.

Initializing a project in the current directory

  • Automatically creates the Cargo.toml configuration file and initializes the current directory as a Cargo project.

Creating a new project

  • Creates a new project using the cargo command.
  • Automatically creates the Cargo.toml configuration file.
  • Automatically initializes a git repository.
  • Automatically creates the src directory and a main.rs file inside the src directory.

<project_name>: Project name

Project configuration file

package: Configuration of the current project.

name: Project name.
version: Project version.
edition: Rust version being used.

dependencies: Configuration of third-party dependent crates.

Compiling a project managed by Cargo

  • When compiled for the first time, a cargo.lock file will be generated in the project’s root directory.
  • The compiled executable file will be generated in the target directory.

Defining variables

Naming conventions for variables

  • Can only contain lowercase letters, uppercase letters, numbers, and underscores.
  • Can only start with a lowercase letter, uppercase letter, or underscore.
  • Variable names are case-sensitive.

Defining an immutable variable

  • An immutable variable cannot be reassigned after it is defined.

Defining a mutable variable

Mutable reference

Variable hiding

  • Defining a variable with the same name will hide the old variable and keep only the results of the last definition.

Hiding a variable and changing its data type

Defining constants

  • Constants are usually named with uppercase letters.
  • The data type must be specified when defining constants.
  • Constants cannot be hidden or redefined.
  • Constants are replaced with specific values during compilation.
