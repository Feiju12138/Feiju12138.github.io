Send complex requests

  • Configure the dio object to send complex requests

baseUrl: Configure the domain part of the URI, so that the domain name can be omitted when sending requests through the dio object and directly access the URN.
headers: Define the request header parameters.
validateStatus: Define which status codes are considered as normal requests. Requests that are not considered normal will throw exceptions and will not automatically parse the request body.
method: Request method.
path: Request path, if baseUrl is specified, the domain part does not need to be specified again.
responseType: Response type

json: Default value
plain: Plain text
stream: Binary stream, used for handling files
queryParameters: Parameters in the request line.
data: Parameters in the request body.
