  • Read-only properties cannot be modified

  • Use the static keyword to define class methods

Constructor

  • Use constructor as the method name to define a constructor
  • this represents the current instance

Create objects using the constructor

Inheritance

Subclass inherits from superclass

  • Use the extends keyword to implement class inheritance

Method overriding

  • If a subclass defines a method with the same name as the superclass, the method in the subclass will override the method in the superclass

Get the instance of the superclass

  • Use the super keyword to get the instance of the superclass
    • Call the constructor of the superclass using super()
      • If a subclass defines a constructor, be sure to call the constructor of the superclass in the subclass’s constructor
    • Call a method of the superclass using super.methodName

Abstract class

Define an abstract class

  • Use the abstract keyword to define an abstract class, abstract classes cannot create objects, they are mainly used as base classes to be inherited

Define abstract methods

  • In an abstract class, abstract methods can be defined using the abstract keyword, abstract methods do not have a method body
  • Subclasses that inherit from an abstract class that contains abstract methods must override the abstract methods

Interfaces

Define an interface

  • Use the interface keyword to define an interface
  • Interfaces are used to define the structure of a class and specify which properties and methods a class should have
  • The methods in an interface can only be abstract methods

Implement an interface

  • Use the implements keyword to implement an interface
  • The subclass that implements an interface must override all the properties and methods defined in the interface

Encapsulation

  • Use access modifiers to limit the scope of a property

public: public, default value, can be modified at any time
private: private, can only be used within the class
protected: protected, can only be used within the current class and its subclasses

Generics with interface inheritance

  • When calling a function that includes a generic that implements an interface, the parameter passed should be an instance of a class that implements the interface
