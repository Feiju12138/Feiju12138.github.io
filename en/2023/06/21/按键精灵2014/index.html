Preface

Learning Notes for AutoHotkey 2014

The scripts of AutoHotkey 2014 are based on VB and can use some built-in VB commands; the scripts of AutoHotkey X are based on Lua.
The file extension of the source code is .Q, but the file not only contains the source code, but also includes file properties, which can be modified and debugged using the AutoHotkey 2014’s editor.

Get the Color Value of the Current Mouse Position

  • Get the 6-digit hexadecimal color value of the current mouse position

Find Color by Position

<x>: x-axis coordinate
<y>: y-axis coordinate
FFFFFF: Hexadecimal color value
0 means the colors are the same, 1 means they are different, 2 means they are similar

Find Color within an Area

  • Find color starting from the top left corner within an area, with a left to right, top to bottom order

<x_start>, <y_start>: Coordinates of the top-left corner of the selection
<x_end>, <y_end>: Coordinates of the bottom-right corner of the selection
intX: Return value, the x coordinate of the found result, if not found, it returns a number less than 1
intY: Return value, the y coordinate of the found result, if not found, it returns a number less than 1

More Configurations

<mod>: Specify the search mode

0: From top left to bottom right
1: From center
2: From bottom right to top left

<similarity>: Specify the color similarity, range [0,1]

Image Conditions

  • Add the image to attachments first

AutoHotkey can only recognize bitmap images in .bmp format.
It cannot find monochrome images.
If the four corners of the image have the same color of pixels, all occurrences of that color in the image are considered transparent.

  • Then reference the image in the code

<filename>.bmp: The name of the image in the attachments

Count the Number of Standard Color Blocks

Similarity: Range [0,1]
Color value: 6-digit hexadecimal color value, sequence is BGR
