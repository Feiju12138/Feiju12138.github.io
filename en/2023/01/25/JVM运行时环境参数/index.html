Introduction

JVM Runtime Environment Parameter Learning Notes

Specify the Size of the Virtual Machine Stack

  • The virtual machine stack stores the Java program’s running threads.
  • Each stack is composed of multiple stack frames, corresponding to the memory used by each method call.
  • Each thread can only have one active stack frame, corresponding to the currently executing method.
  • Virtual machine stack memory: the maximum memory occupied by the threads running in the Java program.
    • The default virtual machine stack memory in Windows is the default virtual memory.
    • The default virtual machine stack memory in Linux/MacOS is 1024KB.
  • Since the total memory of the operating system is fixed, if the virtual machine stack memory is larger (the larger the memory that can be used by a single thread), the smaller the total number of threads that can be created by a process. Therefore, the virtual machine stack memory is not the bigger the better.

-Xss: Specify the virtual machine stack memory.

Initial Size of Heap Memory

Maximum Size of Heap Memory

  • Whenever a new object is created in a Java program, it is placed in the heap memory.
  • The data in the heap memory is shared among threads, so thread safety needs to be considered.
  • The heap memory has a garbage collection mechanism.
  • In order to find heap memory overflow problems, it is usually possible to set the heap memory to a smaller size so that the problems can be exposed early and avoid missing heap memory overflow problems caused by excessive heap memory.

-Xmx: Specify the heap memory.

Survivor Ratio

-XX:SurvivorRatio=8: 8/10 represents Eden, 1/10 represents Survivor From, and 1/10 represents Survivor To

Dynamically Adjust Survivor Ratio

-XX:InitialSurviorRatio=8: Specify the initial survivor ratio
-XX:+UseAdptiveSizePolicy: Enable dynamic adjustment

Promotion Threshold

  • When the promotion threshold is reached, the objects will be promoted from Survivor to the old generation.
  • Some garbage collectors have a default promotion threshold of 6, while others have a default promotion threshold of 15.

Print Promotion Details

Size of Permanent Generation

  • Used to load classes before Java 1.8, occupying the memory of the JVM.

-XX:MaxPermSize=: Specify the size of the permanent generation.

Print GC Details

Enable Minor GC

  • Perform a minor GC before performing a Full GC.

Size of Metaspace

  • Used to load classes in Java 1.8 and later, occupying the memory of the operating system.

-XX:MaxMetaspaceSize=: Specify the size of the metaspace.

Number of StringTable Buckets

  • Optimize the StringTable pool by modifying the number of buckets in the StringTable.
  • The data structure of StringTable is implemented with a hash table (array + linked list), and each array is called a bucket.
    • When maintaining a large number of strings in the string pool, the more buckets, the faster the speed.
    • When maintaining a small number of strings in the string pool, the fewer buckets, the faster the speed.

-XX:StringTableSize=: Specify the number of buckets in the StringTable pool, ranging from [1009, 2305843009213693951]

Disable Explicit GC Invocation

  • Manually invoking GC for garbage collection not only cleans up the young generation but also cleans up the old generation, resulting in a long pause time for the program. Therefore, explicit GC calls are disabled when performing JVM tuning, which means disabling the System.gc() calls in the code.
