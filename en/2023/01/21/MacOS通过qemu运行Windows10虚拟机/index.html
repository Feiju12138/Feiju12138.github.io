【英文】MacOS通过qemu运行Windows10虚拟机

Posted on 2023-01-21 Edited on 2024-03-28 Views:

Introduction

Running Windows 10 virtual machine on MacOS Apple Silicon through qemu

Preparation

Download dependencies required for compiling QEMU

Download QEMU patch

  • Find the latest patch provided by Alexander Graf for Apple Silicon

  • Click on series to download

Download edk2

  • Download the aarch rpm package from the edk2 repository

Download Rufus, a tool for creating system virtual machine

Link

Compile QEMU

Download the project

Apply the patch

~/Downloads/hvf-Implement-Apple-Silicon-Support.patch: QEMU patch for Apple Silicon

Compile the project

Add edk2 package

  • Copy all files from the edk2 package to the root directory of the compiled QEMU

edk2.git-aarch64-0-20220719.209.gf0064ac3af.EOL.no.nore.updates.noarch.rpm: edk2 package

Finish compiling

  • All files under the build directory are the compiled QEMU program

Create Windows virtual machine file (needs to be done on Windows)

  • Create a VHDX virtual disk on Windows, install the Windows system ISO image to the VHDX virtual disk using a tool, as the Windows virtual machine file

Create empty VHDX virtual disk

  • Right click on This PC -> Manage -> Disk Management -> Action -> Create VHD

  • Specify the location and filename of the VHDX file -> Specify the virtual hard disk size -> Choose VHDX format -> Choose Dynamic expanding -> OK

If Dynamic expanding is chosen, the virtual hard disk size can be specified freely

Install Windows operating system using Rufus

  • Specify the virtual disk in Device -> Choose system image file in Boot selection -> Choose Windows To Go in Image option -> Start

  • OK

  • OK

  • OK

Finish creating VHDX

  • Move the created VHDX file to the qemu/build directory

Start the Virtual Machine

  • Start the virtual machine by running the command in the qemu/build directory

<vhdx>: complete filename of vhdx

Initialization

  • On first boot, press Esc key to exit BIOS options -> choose Device Manager and press Return

  • Select OVMF Platform Configuration and press Return

  • Change Change Preferred to 1024x768

  • Save settings with F10 -> Confirm with Y

Conclusion
