Defining Constructors in a Class

  • Constructors are executed immediately when an object is created to initialize the object
  • The constructor has the same name as the class
  • Constructors do not have a return value and do not require the void keyword
Default Constructor
  • By default, the default constructor is used to create an object
  • Each class can only have one default constructor

  • Each class can have multiple named constructors

Private Access

  • Private properties and methods can be created by adding _ as a prefix to the property or method name within the class
  • Classes need to be separated into individual Dart files to implement private access

Getter Methods

Defining Getter Methods

  • Getter methods are marked with get for the return type
  • Getter methods have no parameter list and do not need to use () to define no parameters

Using Getter Methods

  • If a class has defined a getter method, the getter’s return value can be obtained directly by calling the getter method name

Setter Methods

Defining Setter Methods

  • Setter methods are marked with set for the return type
  • The parameter list of the setter method is used to pass the new value of the variable to be modified

Using Setter Methods

Static Properties and Methods

  • Static properties are marked with the static keyword; static methods are marked with the static keyword
  • Static properties and methods can be accessed directly by the class name or through objects
  • The data of static properties and methods is stored in the heap and is shared by all objects
  • Static methods cannot access non-static methods, but non-static methods can access static methods
  • It is recommended to use the syntax propertyName to call static properties in non-static methods in the current class; it is recommended to use the syntax this.propertyName to call non-static properties in non-static methods in the current class

Object Operators

Conditional Operator

  • By using the ? keyword to qualify an object, non-null checks can be performed on the object
    • If the object is null, the method call is not executed
    • If the object is not null, the method call is executed

Type Checking

  • The is keyword is used to check if an object is of a certain type and returns a Boolean value

Type Conversion

  • The as keyword is used to forcibly typecast an object to the specified class

Cascading

  • The .. keyword is used to invoke multiple properties or methods of the same object in succession

Inheritance

  • A subclass can only inherit from one parent class

Constructors

  • A subclass cannot inherit the constructor of the parent class and needs to redefine the constructor
  • The parameters obtained by the constructor of the subclass are usually passed to the super class for assignment

Method Overriding

  • When a subclass overrides a parent class method, the method is marked with the @override annotation on the line above the method declaration

Accessing Properties and Methods of the Parent Class in a Subclass

  • A subclass can access the properties and methods of the parent class using the super keyword

Defining Abstract Classes

  • Abstract classes are marked with the abstract keyword
  • Abstract methods are defined in abstract classes without method bodies
  • Subclasses that inherit from abstract classes must override all properties and methods defined in the parent class
  • Abstract classes can also define regular methods

Polymorphism

  • Allows a pointer of a subclass type to be assigned to a pointer of a parent class type, and the same method call will have different effects

Defining Interfaces

  • Dart does not have interfaces as they are traditionally understood, but instead, an interface is defined using an abstract class
    • An abstract class is usually used to define an interface, where the abstract class defining the interface only defines abstract methods and does not define regular methods
  • Subclasses implement the interface using the implements keyword and must override all properties and methods defined in the parent class
  • Subclasses may use the extends keyword to inherit from another class and use the implements keyword to implement multiple interfaces
