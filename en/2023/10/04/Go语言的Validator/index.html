【英文】Go语言的Validator

Posted on 2023-10-04 Edited on 2024-03-28 Views:

Preface

Go language uses Validator to validate the field values of a structure. The structure variable can only be successfully instantiated if all the field values conform to the validation rules.

Download Dependencies

Initialization

validate.SetTagName(""): Set the tag name

validate: Default value, tag name

Tag Separators

,: Multiple validations are required
|: Only one validation is required
-: Skip the validation of the field

String Validation

alpha: Letters
alphanum: Letters and numbers
alphaunicode: Letters and Unicode characters (including Chinese characters)
alphanumunicode: Letters, numbers, and Unicode characters (including Chinese characters)
ascii: ASCII characters
contains=string: Contains the specified string
excludes=string: Does not contain the specified string
startswith=string: Starts with the specified string
endswith=string: Ends with the specified string
startsnowith=string: Does not start with the specified string
endsnowith=string: Does not end with the specified string
lowercase: All letters in the string are lowercase
uppercase: All letters in the string are uppercase
boolean: Boolean string
multibyte: Multibyte string
number: Integer value string
numeric: Integer value and float value string

Format String Validation

base64: Base64 formatted string
base64url: Base64 formatted URL string
credit_card: Credit card number string
datetime=2006-01-02 15:04:05: String with specified date format
e164: Phone number string with country code, e.g., +8613800000000
email: Email address string
hexadecimal: Hexadecimal string
json: JSON formatted string
jwt: JWT formatted string
longitude: Longitude format string
latitude: Latitude format string
timezone: Timezone format string, e.g., Asia/Shanghai
iscolor: Color string
country_code: Country code string

Network Address Validation

fqdn: Domain name
hostname: Hostname
hostname_port: Hostname and port number
ip: IPv4 address and IPv6 address
ipv4: IPv4 address
ipv6: IPv6 address
mac: MAC address
uri: Uniform Resource Identifier
url: Uniform Resource Locator
url_encode: URL encoded Uniform Resource Locator

Numeric Comparison Validation

eq=0: Equal to
gt=0: Greater than
lt=0: Less than
get=0: Greater than or equal to
let=0: Less than or equal to
ne=0: Not equal to

Required Validation

Required

  • Must not be zero value

required: Required
required_if=field value: Required when the specified field is the specified value
required_unless=field value: Required when the specified field is not the specified value
required_with=field name: Required when the value of the specified field is not zero value
required_with_all=field name 1 field name 2: Required when the values of the specified fields are not zero values
required_without=field name: Required when the specified field is zero value
required_without_all=field name 1 field name 2: Required when all the specified fields are zero values

Not Required

  • Must be zero value

excluded_with=field name: Not required when the value of the specified field is not zero value
excluded_with_all=field name 1 field name 2: Not required when the values of the specified fields are not zero values
excluded_without=field name: Not required when the specified field is zero value
excluded_without_all=field name 1 field name 2: Not required when all the specified fields are zero values

Directory and File Path Validation

dir: Directory path
file: File path

Other Validations

isdefault: Zero value
len: String length
max: Maximum value, can be used for numeric, string, slice, set
min: Minimum value, can be used for numeric, string, slice, set
oneof=<value_1> <value_2>: One of the values in the enumeration
omitempty: Skip the validation of the field if it has no value
dive: Dive into one level of validation, usually used on slice or map type fields

Relative Field Validation

Relative to Other Basic Type Field Validation

eqfield=field name: Equal to the specified field value of the current structure
gtfield=field name: Greater than the specified field value of the current structure
ltfield=field name: Less than the specified field value of the current structure
gtefield=field name: Greater than or equal to the specified field value of the current structure
ltefield=field name: Less than or equal to the specified field value of the current structure
nefield=field name: Not equal to the specified field value of the current structure
fieldcontains=field name 1 field name 2: Contains the specified field values of the current structure
fieldexcludes=field name 1 field name 2: Does not contain the specified field values of the current structure

Relative to Other Structure Type Field Validation

eqcsfield=field name.field name: Equal to the specified field value of the child structure
gtcsfield=field name.field name: Greater than the specified field value of the child structure
ltcsfield=field name.field name: Less than the specified field value of the child structure
gtecsfield=field name.field name: Greater than or equal to the specified field value of the child structure
ltecsfield=field name.field name: Less than or equal to the specified field value of the child structure
necsfield=field name.field name: Not equal to the specified field value of the child structure

Define Tags on a Structure

min=1: Minimum length of a slice
min=2: Minimum length of each string
