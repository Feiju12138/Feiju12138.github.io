Operations on Namespaces

  • namespace can be abbreviated as ns

View namespaces

  • Pods with the same name can exist in different namespaces

Create a namespace

<name>: namespace name

Delete a namespace

  • Delete the namespace and all the pods contained in it

<namespace_name>: namespace name

Operations on Pods

  • pod can be abbreviated as po

  • Kubernetes deploys containers on a per-pod basis

  • Pod is an encapsulated object of a Docker container

  • A pod can encapsulate multiple Docker containers

Operations on Deployment Workloads

  • deployment can be abbreviated as deploy

  • Deployment is a higher-level resource used to deploy or upgrade applications

  • When creating a Deployment, a ReplicaSet resource is created, and the actual Pods are created and managed by the ReplicaSet, not directly by the Deployment

  • Deployment can introduce another ReplicaSet during the application rolling upgrade process and coordinate the two ReplicaSets

Operations on Services

  • service can be abbreviated as svc

  • Services control access to containers

  • Provide an invariant access address that can forward calls to all containers, similar to Spring Cloud Zuul

  • Implement service discovery and load balancing

Expose a port to other containers

  • Ports exposed in this way can only be accessed by other containers in Kubernetes

<container_name>: container name
--port=80: Configure the externally exposed port
--target-port=80: Configure the port forwarded to the container
-n <namespace>: Specify the namespace
--type=: Type of exposure

ClusterIP: Default, internally exposed
NodePort: Externally exposed

  • Access from other containers

    • Other containers can access by IP
    • Other containers can access by domain name
      • The default domain name format is: <Deployment_name>.<Namespace_name>.service:80

  • Check the Service to get the port number exposed internally

  • Access the exposed port from other containers
