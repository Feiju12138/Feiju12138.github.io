<src>: File path, an error will be reported if the file does not exist

Open Modes and Permission Control

<src>: File path, an error will be reported if the file does not exist
<mode>: Modes for opening the file, multiple modes can be specified using | as a separator

os.O_RDONLY: Open the file in read-only mode
os.O_WRONLY: Open the file in write-only mode
os.O_RDWR: Open the file in read-write mode
os.O_APPEND: Append content to the file
os.O_CREATE: Create a new file if it does not exist
os.O_EXCL: Report an error if the file does not exist
os.O_SYNC: Open the file for asynchronous I/O
os.O_TRLNC: Open the file and truncate it immediately

<jurisdiction>: Permission control, only valid in Unix-like operating systems

0777: 0 represents octal, 777 represents read (4), write (2), and execute (1) permissions for owner, group, and others respectively

Open Multiple Modes

<mode>: Multiple modes are specified using | as a separator

Read Content Line by Line

Read the Entire File Content at Once

  • Read the entire file at once using an unbuffered Reader
  • Suitable for reading small files
  • Returns a []byte
  • Because ioutil.ReadFile() does not explicitly call Open() to open the file, there is no need to explicitly call Close() to close the file, as Close() is already encapsulated
