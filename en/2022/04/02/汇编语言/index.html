Preface

Assembly language study notes

Preparation

  • edit.com: editor
  • masm.exe: assembler
  • link.exe: linker
  • debug.exe: debugger

DOSBox

  • Virtual 16-bit 8086 environment

Mount real disk and switch

  • Operate directly in DOSBox

c: virtual drive letter after mounting
<dir>: directory path in the real environment, cannot contain Chinese

Add mount and switch to startup options

  • The configuration file on Windows is in the software root directory (C:\Program Files (x86)\DOSBox-0.74-3\), double-click the startup options batch file (DOSBox 0.74-3 Options.bat) to automatically open the configuration file with Notepad (C:\Users\xxx\AppData\Local\DOSBox\dosbox-0.74-3.conf), and append the configuration at the end of the file
  • Configuration file path on MacOS: ~/Library/Preferences/DOSBox 0.74-3 Preferences

Full screen

  • Windows: alt+enter
  • MacOS: option+return

Usage of edit.com

Launch edit

Save

  • alt+f->save

Edit a file when launching edit

  • If the file exists, edit directly.
  • If the file does not exist, create a new file and start editing.

<name>: filename

Edit assembly program

  • Create a .asm assembly source file

Comment

  • Write comments after ;

Segment definition

  • The first step in creating an assembly program is to define segments, which tell the operating system how many segments the program needs (up to 4 segments can be defined at most).

  • When defining a segment, the segment name needs to be specified. In fact, the segment name is the symbolic address of the segment address.

    • The type of segment is determined by the segment association and segment address transfer, not the segment name itself.

  • Segment name naming rules

    • Only letters, numbers, and underscores are allowed
    • Cannot start with a number
    • Characters cannot exceed 8 characters, and if more than 8 characters, the trailing characters will be discarded

  • Segment name naming conventions

    • As long as it meets the naming rules, the segment name can be defined arbitrarily
    • The code segment is usually named code
    • The data segment is usually named data
    • The stack segment is usually named stack

Segment association

  • The segment association statement must be placed on the first line after the segment definition instruction

Segment address transfer

  • Except for the code segment, other segments need to be transferred before the segment type can be determined

  • Use the mov instruction to transfer the segment address, but mov does not allow immediate values to be stored in registers, so a general register is needed

Symbolic address

  • Write a label before the instruction. The label is a symbol that points to the address where the instruction is stored, known as the symbolic address.
  • Labels can be placed immediately before the instruction or above the instruction.
  • Not every instruction needs to be preceded by a label. Labels are only written when the memory address needs to be obtained. The first instruction of the program must have a label before it.
  • Before the start of code execution, write the start label, and after the execution ends, write the end label. The start label is usually named start.

Instruction

  • Pseudo-instruction: instructions that do not have specific operands
  • Instruction: instructions with specific operands
    • Double operand instruction
      • instruction destination operand, source operand
    • Single operand instruction
      • instruction destination operand
      • instruction source operand
    • No operand instruction
      • instruction

Code segment

  • The code segment is usually placed after all segments.
  • The code segment does not require segment address transfer.

Data segment

  • The data segment requires both segment association and segment address transfer in the code segment.
  • When data needs to be stored, the data segment can be defined.

Define data

  • Data definition pseudo-instructions

    • Byte data definition pseudo-instruction
      • Byte data pseudo-instructions use the db keyword to allocate 1 byte of space each time
    • Word data definition pseudo-instruction
      • Word data pseudo-instructions use the dw keyword to allocate 2 bytes of space each time
    • Double-word data definition pseudo-instruction
      • Double-word data pseudo-instructions use the dd keyword to allocate 4 bytes of space each time

  • Principles for storing multi-byte data

    • The high byte is stored at the high address
    • The low byte is stored at the low address

Get data

  • Define the symbolic address before the data, and the data can be obtained through the symbolic address. The symbolic address can customize the name.
