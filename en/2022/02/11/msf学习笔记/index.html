Foreword

The Metasploit Project is a computer security project aimed at providing security vulnerability information. It assists security engineers in conducting penetration testing and developing intrusion detection system signatures. (Wikipedia)

This article is only for learning network information defense

Development History

  • October 2003: HD Moore developed Metasploit v1.0 using Perl language.
  • April 2004: With the help of Spoonm, the code of Metasploit was completely rewritten. Main updates: 18 attack modules and 27 attack payloads, and it provided three interfaces for use: console terminal, command line, and web interface.
  • August 2004: HD and Spoonm gave a speech at the BlackHat Global Developer Conference held in Las Vegas with the newly released Metasploit v2.0.
  • 2005: At CanSecWest and hacker conferences, HD and Spoonm pointed out that the Perl version of Metasploit had many problems. After 18 months, the Metasploit team completely rewrote Metasploit in Ruby language and released Metasploit v3.0. Main updates: 177 attack modules, 104 attack payloads, and 30 auxiliary modules.
  • October 2009: Metasploit was acquired by Rapid7.
  • October 2010: Rapid7 released commercial versions: Metasploit Express and Metasploit Pro.
  • August 2011: The Metasploit team released v4.0. Main updates: introduced post-exploitation modules (used for long-term authorization positions and attack achievements extension).
  • January 2019: The Metasploit team released v5.0. Main updates: introduced new modules evasion capabilities (used to bypass anti-virus software).

Technical Architecture

  • Auxiliary Modules: Provide support for information collection in penetration testing, including scanning and checkpoints for various network services, building false services to collect passwords, password guessing and cracking, sensitive information sniffing, probing for sensitive information leakage, fuzz testing to discover vulnerabilities, and implementing network protocol deception, etc.
  • Exploitation Modules: Exploit discovered security vulnerabilities or configuration weaknesses to attack target systems, implant and execute attack payloads, and obtain remote access to the target system.
    • Active Exploitation: The vulnerabilities exploited by active exploitation are located in server software and upper-layer application programs of the service carrier.
    • Passive Exploitation: The vulnerabilities exploited by passive exploitation are located in client software, such as browsers, browser plugins, email clients, etc.
  • Payload Modules: Run a piece of implanted code on the target system after successful exploitation, usually used to open a control session connection on the target system for penetration.
  • NOP Modules: Some NOP instructions or unrelated operation instructions that do not have any substantial impact on program operation. For example, the opcode is 0x90 on an x86 architecture CPU host.
  • Encoder Modules: Ensure that no bad characters that should be avoided during the penetration process appear in the attack payload; perform bypass of anti-virus software for the attack payload.
  • Post-Exploitation Modules: Provide attack springboard.
  • Evasion Modules: Perform bypass of the payload.
