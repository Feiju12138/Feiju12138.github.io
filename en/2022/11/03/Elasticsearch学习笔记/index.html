Preface

Elasticsearch Learning Notes

Concepts

  • Document: Each piece of data is a document.
  • Term: The words separated according to semantics in the document.
  • Index: A collection of documents of the same type.
  • Mapping: The constraint of fields in the index on the document, similar to the structural constraints of a database table.

Concepts in Elasticsearch

Comparison of Concepts between MySQL and Elasticsearch

Concept in Elasticsearch Description Concept in MySQL
Index A collection of documents of the same type. Similar to a data table in MySQL. Table
Document Each piece of data is a document, and documents are formatted as JSON strings. Similar to a row in MySQL. Row
Field Each key in the JSON document. Similar to a column name in MySQL. Column
Mapping The constraint on the values of each key in the JSON document. Similar to the table structure in MySQL. Schema
DSL JSON-style request statements used to manipulate Elasticsearch to implement CRUD operations. Similar to SQL statements in MySQL. SQL

IK Analyzer

  • The IK Analysis plugin integrates the Lucene IK analyzer (/302.html?target=http://code.google.com/p/ik-analyzer/) into Elasticsearch, supporting custom dictionaries.(Github)

Adding Extended Words

  • In addition to the built-in dictionary, the IK Analyzer can also segment words based on the extended dictionary.

  • Create a /plugins/ik/config/ext.dic file, with each line specifying an extended word.

Adding Stop Words

  • If a stop word dictionary is defined, the IK Analyzer will not segment words when encountering extended words.

  • Create a /plugins/ik/config/stopword.dic file, with each line specifying a stop word.

Tokenization Operation

"analyzer": "": Specify the tokenization method.

standard: Standard method. Tokenize word by word.
chinese: Built-in Chinese tokenization. Tokenize word by word.
ik_smart: IK analyzer with minimum words.
ik_max_word: IK analyzer with maximum words.

"text": "": Specify the content to be tokenized.

Autocomplete

completion: Allows fuzzy search using prefix keywords.

index: "": Whether to create an inverted index. An inverted index is only needed if the field participates in searching; if not, the inverted index does not need to be created.

true: Default value, create an inverted index.
false: Do not create an inverted index.

analyzer: "": Specify which analyzer to use.

standard: Standard method. Tokenize word by word.
chinese: Built-in Chinese tokenization. Tokenize word by word.
ik_smart: IK analyzer with minimum words.
ik_max_word: IK analyzer with maximum words.

properties: {"": "", "": ""}: Subfields of the field.
copy_to: "other_field_name": Copy the value of the current field to other fields (actually, an index is created instead of actually copying the value), so that when searching other fields, the value of the current field will also be searched.

Specifying a Custom Analyzer

  • After reconfiguring the pinyin analyzer as a filter, combine it with the IK analyzer to create a new custom analyzer.
  • When creating an index, specify the custom analyzer to be used when adding a new document, and specify the IK analyzer to be used when searching for a document.

Character Filter: Process the original document between tokenizations.
tokenizer: Divide the text into terms according to the specified rules or tokenizer. Specify keyword if no tokenization is required.
filter: Process the terms output by the tokenizer using the specified filter or tokenizer.

"analyzer": "": Specify the analyzer used when creating the index.
"search_analyzer": "": Specify the analyzer used when searching.
