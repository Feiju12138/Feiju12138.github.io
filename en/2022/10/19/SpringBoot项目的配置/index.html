Preface

Configuration of SpringBoot project

Types of Configuration Files

application.properties: Configuration in key-value format (default)
application.yml: Configuration in YAML format (most commonly used). There is no difference between .yml and .yaml extension.
application.yaml: Configuration in YAML format. There is no difference between .yml and .yaml extension.

  • When all three configuration files exist, the priority is application.properties > application.yml > application.yaml

Manually Configuring SpringBoot Project Configuration Files

  • If IDEA does not automatically recognize the configuration files, there may be problems with the editor not prompting. You can manually add configuration to the SpringBoot project configuration files configuration.

  • File->Project Structure

  • Facets->Select SpringBoot module->Select configuration files configuration button->Select add button->Select the file that has not been added to the SpringBoot configuration file configuration->Open->OK->OK

Configuration Options

Server Port Configuration

SpringBoot Log Level Configuration

info: Default value, only display information level
warn: Warning level (less information)
debug: Debug level (more information)

Configuration for Multiple Environments

Properties Configuration File for Multiple Environments

  • Create multiple configuration files and add the environment name after -
  • Set the enabled environment in the main configuration file

Yaml Configuration File for Multiple Environments

  • Separate multiple sets of configuration files with ---
    • Set the enabled environment in the first section. The first section is common and will not be modified due to the environment switch.
    • Configure different environments in other sections.

spring.profiles.active: Set the enabled environment based on the environment name
spring.profiles: Specify an environment name and configure all the configurations for the current environment below it.

The above syntax is deprecated and should be replaced with the following syntax

Specifying Environment on Startup

<file>.jar: Specify the jar file to start
--spring.profiles.active=: Specify the environment name

Possible Issues

  • After configuring multiple environments, it is better to execute the clean lifecycle before executing the package lifecycle.
  • Chinese characters in the configuration files may cause packaging failure. You should change the character encoding used by IDEA for packaging to UTF-8 before packaging.

IntelliJ IDEA->Preferences->Editor->File Encodings

Global Encoding: UTF-8
Project Encoding: UTF-8
Default encoding for properties files: UTF-8

Compatibility between SpringBoot and Maven for Multiple Environments

  • In SpringBoot, multiple environments can specify the project configuration for multiple environments.
  • In Maven, multiple environments can specify the dependencies for multiple environments.
  • In general, to avoid conflicts between the two environments, SpringBoot should obtain the environment configuration information from Maven, thereby changing the multiple environments configured by SpringBoot, and achieving the purpose of jointly maintaining the multi-environment configuration of SpringBoot and Maven.

Adding Plugins

  • Add the maven-resources-plugin plugin to allow the custom properties defined in Maven to be accessed by the configuration files in the resources directory.

Configuration in Maven

  • Add custom properties in Maven’s multiple environment configuration to pass to the SpringBoot configuration file.

Configuration in SpringBoot

  • Get the value of the custom property passed from Maven in the SpringBoot configuration file, and dynamically change the environment name of SpringBoot based on the environment name passed in from Maven.

${profile.active}: Get the custom parameter passed from Maven
