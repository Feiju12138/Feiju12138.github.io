Access Modifiers

  • Access modifiers can be added before attributes or methods.
  • Once an access modifier is defined, it applies to all attributes and functions that come after it.

struct and class

  • struct and class have different default access permissions:
    • struct has a default access permission of public.
    • class has a default access permission of private.

Encapsulation

  • Make member attributes private and member functions public.

Object Initialization and Cleanup

  • Define a constructor to initialize an object.
  • Define a destructor to clean up an object.
  • If a class does not define a constructor or destructor, the compiler will provide an empty constructor and destructor.

Constructor

Define a Constructor

  • The constructor is automatically executed when an object is created.
  • The constructor does not have a return value and does not use void.
  • The constructor has the same name as the class.
  • The constructor can have a parameter list, so it can be overloaded.

Calling a Constructor

  • The constructor is automatically called when an object is created.
  • Notes:
    • Do not use an empty pair of parentheses when using the parentheses method, as it will be misinterpreted as a function declaration.
    • Anonymous objects are immediately destroyed after the current line, so the copy constructor cannot be used to initialize anonymous objects. The compiler will convert ClassName(ObjectName) to ClassName ObjectName.

Destructor

  • The destructor is automatically executed before an object is destroyed.
  • The destructor does not have a return value and does not use void.
  • The name of the destructor is prefixed with ~ before the class name.
  • The destructor does not have a parameter list, so there is no method overloading.

Calling Times of Copy Constructor

Default Constructor

  • There is a default function that exists whenever a class is defined:
    • Default constructor: it has no parameters and an empty function body.
    • Default destructor: it has an empty function body.
    • Default copy constructor: it copies the values of member attributes.
  • If a parameterized constructor is defined, C++ will not provide a default constructor, but will provide a copy constructor.
  • If a copy constructor is defined, C++ will not provide a default constructor or other constructors.

Shallow Copy and Deep Copy

  • If there are no pointers in the copied attributes, shallow copy can be used.
  • If there are pointers in the copied attributes, deep copy needs to be manually defined and used.

Shallow Copy

  • The default copy constructor provided by the compiler is shallow copy, which copies the values of member attributes.
  • When there are pointers in the copied attributes, shallow copy only copies the pointer data without creating new space in the heap memory. The problem with this is that when the pointer is freed, there will be errors of repeatedly freeing the heap memory.

Deep Copy

  • Deep copy requires defining a new copy function, which will create a new space in the heap for copying.
  • When there are pointers in the copied attributes, deep copy will create a new space in the heap, so it stores a new pointer.

Initialization List

  • Initialization list is another way to assign initial values to member attributes.

Class Objects as Class Members

  • One class object can be used as a member attribute of another class.

Static Members

  • Adding the static keyword before a member attribute or member function creates a static member.
  • Static member variables are also subject to access permissions.

Static Member Variables

  • All objects share the same data.
  • Memory is allocated during the compilation phase.
  • Declare in the class and initialize outside the class.
