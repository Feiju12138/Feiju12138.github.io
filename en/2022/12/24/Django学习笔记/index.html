Modules in Django

  • Model: Data layer, handles all data-related operations
  • View: View layer, processes user requests
  • Template: Template layer, renders HTML templates through view functions to generate dynamic front-end pages
  • Url: Website entry point, maps URLs to corresponding view functions to access the website
  • Forms: Forms for submitting data entered in browsers and validating the data
  • Admin: Django comes with an administration site for managing submitted data
  • Settings: Django configuration file

Global Configuration in Django

  • <project_name>/<project_name>/settings.py

BASE_DIR: Returns absolute path of the project directory
SECRET_KEY: Key for data encryption, prevents cross-site attacks
DEBUG: Indicates whether the website is in development mode, default value is True, should be set to False before deploying online
ALLOWED_HOSTS: Whitelist of allowed website access, set to ['*'] to allow any website
INSTALLED_APPS: Application registry
MIDDLEWARE: Middleware
ROOT_URLCONF: Base routing configuration for the website
TEMPLATES: Configures HTML static files
WSGI_APPLICATION: Configuration for the development server
DATABASES: Database configuration
AUTH_PASSWORD_VALIDATORS: Encrypts user passwords
LANGUAGE_CODE: Default language of the website

en-us: Default value, English
zh-hans: Simplified Chinese

TIME_ZONE: Default time

UTC: Default value, Coordinated Universal Time
Asia/Shanghai: Shanghai time

USE_I18N: Enables multilingual support
USE_TZ: Set to False
STATIC_URL: Path for static resources
