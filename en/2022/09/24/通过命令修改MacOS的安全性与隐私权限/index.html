【英文】通过命令修改MacOS的安全性与隐私权限

Posted on 2022-09-24 Edited on 2024-03-28 Views:

Preface

Recently, I have been learning Python office automation and have been using the xlwings module. This module allows Python code to manipulate Microsoft Excel. However, it is strange that it works fine on Windows but not on macOS. It returns an error message: MESSAGE: The user has declined permission.

First Exploration

  • Like many ordinary users, I directly copied and pasted my error message into a search engine. Based on the experience of previous users, I found that the reason was that IDEA did not have permission.

Attempt to Resolve the Issue

  • In order to make Python with the xlwings module work correctly, you can try the following methods:
  1. Downgrade your development tool or switch to another one.

It is said that a lower version of IDEA can pop up a permission request window, and switching to VSCode can also solve this problem. However, I didn’t try it because I don’t want to downgrade or switch my development tool.

  1. Run Python code in the terminal.

After testing, this method does work because the terminal can pop up a permission request dialog. Once permission is granted, the code can run normally.


However, this method only solves the problem of running the code. I still cannot run Python code in IDEA. It wouldn’t be elegant for me to write code in IDEA and then run it in the terminal.

  1. Run IDEA in the terminal.

In fact, the principle of this method is the same as the previous one. It just changes the scenario from IDEA requesting control of Excel to the terminal requesting control of Excel. Since the terminal can pop up a permission request dialog, this method indirectly solves the problem. However, I cannot start IDEA in the terminal every time. It is still not elegant.

Summary

  • Through testing, I found that in the Security & Privacy settings, there are two configuration options related to the terminal under “Automation” settings. This means that if I grant IDEA the “Automation” permission, I can use IDEA to operate Excel. However, the “Automation” permission cannot be manually added. It can only be called by the App. I can’t modify the IDEA source code to fix this issue of the permission request window not popping up, especially since IDEA is not open source. So, the exploration must continue.

Second Exploration

  • If I cannot solve the problem myself, I can seek help from others. Since the instructor in the tutorial video I watched was also using macOS and did not encounter this error, it means that the instructor has already solved this problem. So, I bravely asked the instructor about this problem, but the answer I got was: “The instructor uses PyCharm, while I use IDEA. The instructor did not encounter any issues on PyCharm. How about I also switch to PyCharm?”

To sum up, it is the same as the first method in the first exploration. In my heart, I know that it is not caused by the development tool. Various signs indicate that it is caused by macOS permissions. Although changing the development tool may solve the problem, I still want to continue exploring and find the most perfect solution in my opinion.

Third Exploration

  • Since I always believe that it is caused by the macOS system, I decided to reach out to Apple customer service at 4006668800.

As you might expect, Apple customer service is not Apple developers. Their technical level is higher than that of ordinary users, but they can only provide solutions found in the official Apple documentation. So, I did not get the solution I was looking for.

Another Attempt to Resolve the Issue

  • Through the previous explorations, I found the following commands:
  • By observing, I found that the command for viewing all permissions in Security & Privacy actually calls sqlite3 with administrator privileges. I have learned sqlite3 on my own, and since it is a relational database, SQL statements can be used. Therefore, /Library/Application\ Support/com.apple.TCC/TCC.db is the path to the database file. Since this is the case, can I modify the database using the UPDATE statement in SQL?

I connected to the sqlite3 database using the database plugin in IDEA. The method of operation is: simply drag the /Library/Application\ Support/com.apple.TCC/TCC.db database file into IDEA, and IDEA will automatically configure it. You need to download the sqlite3 driver for the first use.
