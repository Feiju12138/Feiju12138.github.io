Preface

Notes on learning C++ Standard Template Library (STL).

Basic Concepts of STL

  • STL is broadly divided into: containers, algorithms, and iterators.
  • Containers and algorithms are seamlessly connected through iterators.
  • Almost all code in STL uses class templates or function templates.

6 Components of STL

  • Containers: various data structures. Examples include: vector, list, deque, set, map.
    • Sequential Containers: emphasize value ordering. Each element in the sequential container has a fixed position.
    • Associative Containers: binary tree structure. There is no strict physical order relationship between elements.
  • Algorithms: various common algorithms. Examples include: sort, find, copy, for_each.
    • Mutating Algorithms: during the operation, the content of the elements in the interval is changed. Examples include: copy, replace, delete.
    • Non-mutating Algorithms: during the operation, the content of the elements in the interval is not changed. Examples include: search, count, traverse, find extreme value.
  • Iterators: act as the glue between containers and algorithms.
    • Input Iterators: read-only access to data, support ++, ==, !=.
    • Output Iterators: write-only access to data, support ++.
    • Forward Iterators: read and write operations on data, and can advance the iterator forward, support ++, ==, !=.
    • Bidirectional Iterators: read and write operations on data, and can move the iterator forward and backward, support ++, --.
    • Random Access Iterators: read and write operations on data, can access any data in a skip-like manner, support ++, --, [n], -n, <, <=, >, >=.
  • Functors: behave like functions and can be used as strategies for algorithms.
  • Adapters: something used to modify the interface of a container, functor, or iterator.
  • Space Allocator: responsible for space allocation and management.

  • Algorithm: for_each
