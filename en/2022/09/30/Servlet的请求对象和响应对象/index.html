Get All Parameter Values Based on a Specified Parameter Name
  • If there are multiple request parameters with the same name, all request values will be returned as a string array
Solving Chinese Garbled Characters in Request Parameters
  • Before Tomcat 8, the default URL decoding method is ISO-8859-1, while the front-end URL encoding method is UTF-8, resulting in Chinese garbled characters
  • Starting from Tomcat 8, the default URL decoding method used is UTF-8, so there is no need to solve the Chinese garbled characters
For GET or POST Requests
  • First convert the ISO-8859-1 data to a byte array, and then convert the byte array to a string based on the UTF-8 coding

value: Parameter value with Chinese garbled characters obtained through req.getParameter("")

For POST Requests
  • Because the processing method of POST request parameters is character input stream, just set the character encoding of the character input stream to UTF-8 before processing the request parameters to solve Chinese garbled characters

Byte Input Stream

Request Forwarding

  • When forwarding a request, the client’s address bar does not change
  • Request forwarding can only forward to server internal resources
  • The whole process undergoes 1 request and 1 response
  • When forwarding a request, data can be carried via request

<url>: Target resource path for request forwarding, no need to add virtual directory

resp: Response object

Redirection

  • When redirecting, the client’s address bar changes
  • Both server internal resources and external resources can be redirected when redirecting
  • The whole process undergoes 2 requests and 2 responses
  • When redirecting, data cannot be carried via request

<url>: Target resource path for response redirection, needs to add virtual directory
