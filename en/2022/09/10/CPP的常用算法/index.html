Search Algorithm

find Searches for Elements in the Container

  • Searches for the specified element in the container. If it exists, returns the iterator of the specified element; if not, returns the iterator end.

Searching for Built-in Data Types

Searching for Custom Data Types

  • Searching for elements of custom data types requires overloading the == operator in the class.

find_if Searches for Elements Based on Condition in the Container

  • Searches for specified elements based on conditions in the container. If it exists, returns the iterator of the specified element; if it does not exist, returns the iterator end.

Searching for Built-in Data Types

Searching for Custom Data Types

adjacent_find Searches for Adjacent Duplicate Elements in the Container

  • Searches for adjacent duplicate elements in the container. If they exist, returns the iterator of the first occurrence of the duplicate element; if not, returns the iterator end.

binary_search Uses Binary Search to Find Elements in the Container

  • Searches for the specified element in the container. If it exists, returns true; if not, returns false.
  • This method cannot be used in unordered sequences.

Sorting Algorithm

sort Sorts the Elements in the Container

Default Sorting Order

  • The default sorting order is ascending.

Sorting Order for Built-in Types

  • Sort in descending order

Custom Sorting Order

random_shuffle Shuffles the Elements of the Container in a Specified Range

  • Shuffles the positions of the elements in the specified range of the container

merge Merges the Elements of Two Containers

  • Merges the elements of two containers and stores them in a new container.
  • The two sequences before merging must be in order.
  • The merged sequence is also in order.
