【英文】JavaWeb的Session

Posted on 2020-05-13 Edited on 2024-03-28 In Java后端学习指北 Views:

Introduction

Session: In computers, especially in web applications, it is called “session control”. The Session object stores the properties and configuration information required for a specific user session. This means that when a user navigates between web pages of an application, the variables stored in the Session object will not be lost, but will persist throughout the entire user session. When a user requests a web page from an application, if the user does not have a session yet, the web server will automatically create a Session object. When the session expires or is abandoned, the server will terminate the session. One of the most common uses of the Session object is to store user preferences. For example, if a user indicates that they do not like to view graphics, this information can be stored in the Session object. For more detailed information on using the Session object, please refer to the “Managing Sessions” section of the “ASP Applications”. Note that session state is only preserved in browsers that support cookies. (Baidu Baike)

Principle

  • Session is implemented based on cookies. When a Servlet stores a Session, Tomcat will automatically pass the Session’s id as a cookie (Set-Cookie: JSESSIONID=<id>) to the client for storage. When the next request retrieves the Session’s id from the cookie, the server will retrieve the data from the Session based on the id.

  • The default expiration time for the cookie that carries the Session id is not set. When the browser is closed, the cookie will be destroyed. Therefore, Session is not suitable for long-term data storage.

Obtaining the Session Object

  • Since Session is automatically created by the server, there is no need to create a Session object.

Overloaded Methods

  • If it exists, return the Session object.
  • If it doesn’t exist, create and return the Session object.
  • If it exists, return the Session object.
  • If it doesn’t exist, return null.

Setting Session Expiration Time

  • The default expiration time for a Session is 30 minutes. If the value is -1, it means it never expires (under the condition that the browser is not closed).

<num>: Unit: seconds

Uniformly Setting Session Expiration Time

  • Set the Session expiration time in the web.xml configuration file, with a default value of 30.

Destruction of Session

Automatic Destruction

  • By default, if a session is not accessed for more than 30 minutes, it will be automatically destroyed. (30 minutes is the default time, which can be modified but not recommended)

Manual Destruction

  • When the invalidate method of the session is called, the session will be immediately destroyed.

Normal Server Shutdown

  • Hibernation (serialization): When the server shuts down normally, before closing, the server will serialize the internal session object and save it as a file in the server’s work directory, becoming a file.
  • Activation (deserialization): When the server starts up again, the hibernated session will return to the server and become an object in the server.

Unexpected Termination

  • When the server is shut down abnormally (hardware damage, power failure, memory overflow, etc.), the session will be destroyed along with the server’s shutdown.
