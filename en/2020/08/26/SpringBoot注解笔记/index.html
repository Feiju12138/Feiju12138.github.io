Preface

SpringBoot Annotation Knowledge Note

Entry Annotation

  • @SpringBootApplication
  • Describes the startup class of the SpringBoot project. The main method is defined in the class, which reads the configuration file at runtime, loads the specified resources, and performs initialization operations.

Add Package Scanning

  • @ComponentScan(“”)

  • Describes the startup class of the SpringBoot project.

  • Defines the package name as a parameter and defines the classes in the specified package and subpackage as beans.

  • @MapperScan(“”)

  • Defines the package name as a parameter and tells the Mybatis framework to create proxy objects for all interfaces in the specified package.

Define ordinary components

  • @Component
  • Can obtain and inject objects from the bean pool through the @Autowired annotation in the business class.
  • If the bean name is not specified, it defaults to the lowercase first letter of the class name (provided that the naming convention is standardized).

SpringMVC

Persistence layer

  • @Mapper
  • Marks the Dao layer, a special type of @Component.

Service layer

  • @Service
  • Marks the service layer, a special type of @Component.

Controller layer

  • @Controller
  • Marks the controller layer, a special type of @Component.

Describe the implementation class object of the data layer

  • @Repository
  • Describes on the class, a special type of @Component.

Respond to clients

  • @ResponseBody
  • When this annotation is used to describe methods in the controller layer, it is used to tell the Spring framework that the return value of this method can be converted into a specific format (such as JSON) and written to the response body of the response object.
  • The return value of the method is not encapsulated in the ModelAndView object, and will not be parsed by the resolver, but directly responds to the client based on the response object.

RestController

  • @RestController
  • Equivalent to simultaneously describing @Controller and @ResponseBody

RestControllerAdvice

  • @RestControllerAdvice
  • Equivalent to simultaneously describing @ControllerAdvice and @ResponseBody

Handle requests

Handle all requests

  • @RequestMapping(“”)
  • The parameter is the link address to jump to.
  • Can describe classes or methods.

Define request methods

  • @RequestMapping(value=””,method=RequestMethod.GET)
  • @RequestMapping(value=””,method=RequestMethod.POST)

Handling Get requests

  • @GetMapping()
  • Equivalent to @RequestMapping(method=RequestMethod.GET)

Handle POST requests

  • @PostMapping()
  • Equivalent to @RequestMapping(method=RequestMethod.POST)

Allow any website to access via JavaScript

  • @CrossOrigin
  • Marked on the class or method

Specify parameters

  • @Param(“”)
  • Describes the parameter and can specify the parameter.

Rest-style parameter passing

  • @PathVariable(“”)
  • Describes the parameter and can specify a REST-style parameter.
  • If the parameter name in the parameter list is the same as the parameter name in the link, @PathVariable can be omitted.

Specify request parameters

  • @RequestParam
  • Describes the parameter and specifies the annotated object as the request receiving object.
  • Used to receive application/x-www-form-urlencoded format data.

Support for unmatched number of parameters

  • @RequestParam(required=false)
  • By default, required is true, which means that the number of parameters must match, otherwise an error will be reported.

Specify request parameters

  • @RequestBody
  • Describes the parameter and specifies the annotated object as the request receiving object.
  • application/json and application/xml format data must be received with @RequestBody.
  • Can also be used to receive application/x-www-form-urlencoded format data.

DI inject object

  • @Autowired
  • Describes the attribute and can obtain and inject objects from the bean pool. The premise is that the implemented class or inherited class is annotated with Spring’s component annotations.
  • Describes the constructor method can be omitted.
  • Describes the set method, usually with a parameterless constructor.
  • The parameterized constructor is used independently.

Specify the injected object name

  • @Qualifier(“”)
  • Needs to be used with @Autowired.
  • The object name is in lowercase.
  • Can describe attribute, constructor method, and set method parameters.

Specify the primary

  • @Primary
  • When there are multiple objects with the same name, the object marked with @Primary annotation is the preferred object.

Test class

  • @SpringBootTest
  • Must be marked on the test class of the SpringBoot project.
  • The methods in the class are marked with @Test to perform unit testing using Junit.

Global exception handling class

  • @ControllerAdvice
  • Describes a control layer global exception handling class. In this class, exception handling methods can be defined to handle exceptions.

Exception handling methods

  • @ExceptionHandler(Throwable.class)
  • The method marked with this annotation is an exception handling method. The exception type defined inside the annotation is the type of exception that this method can handle, including subclass types of exceptions.
  • You can pass a parameter of the same type (or superclass type) as the exception handling method in the parameter list.

Configuration class

  • @Configuration
  • Describes a configuration class marked by this annotation.

Spring-managed Bean

  • @Bean
  • When the entire third-party Bean object is managed in the Spring container, the creation of the Bean object can be placed in a method, and then described using the @Bean annotation. When the Spring container manages the Bean object, it will use the method name as the key to store the Bean object.

Lazy loading

Marked on the class

  • @Lazy(value=true)
  • Abbreviated as @Lazy(true)
  • Abbreviated as @Lazy
  • Also known as lazy loading
  • This annotation tells the Spring framework that if the instance of the class it describes is temporarily not needed, don’t create it first, create it when it needs to be called.
  • Only effective in singleton cases

Singleton and prototype

  • Marked on the class

Singleton

  • @Scope(“singleton”)
  • If @Scope parameter is not written or @Scope is not written, it defaults to singleton scope.
  • Singleton puts the created object in the bean pool.
  • Each class will only create one object, which can be called repeatedly.

Prototype

  • @Scope(“prototype”)
  • Prototypes are created when the object is used and destroyed after use.
  • Each class can create multiple objects for multiple instances.
  • After this instance is created, it will not be managed by spring. Spring can initialize it, but it is not responsible for destruction.

Object lifecycle methods

Initialization method

  • @PostConstruct
  • Marked on the method
  • The described method will be executed after the object is constructed
  • Used to perform some initialization operations

End method

  • @PreDestroy
  • The described method will be executed before the singleton object is destroyed
  • Only applicable to singletons
  • The Spring container will remove the object in the container (Bean) before removal. If the object defines a lifecycle destruction method, the lifecycle destruction method of the object will also be called at this time (do some resource release operations in the method).

Exception handling

  • @ControllerAdvice
  • Describes the class as a global exception handling class
  • The class can define exception handling methods to handle exceptions based on these exception handling methods

Global exception handling method

  • @ExceptionHandler(RuntimeException.class)
  • Exception handling method of the global exception handling class

Start asynchronous operation

  • @EnableAsync
  • Marked on the startup class

Configure asynchronous operations

  • @Async
  • The method described by this annotation is an asynchronous join point method
  • This method runs in a separate thread provided by the Spring framework when it is executed
  • If the return value is not void, additional processing needs to be done

Cache

Supporting Cross-Origin Access

  • @CrossOrigin
  • Marked on methods in the Controller
  • Enables cross-origin access to the current URL

Assigning values to objects through configuration files

  • @Value(“${father.son}”)
  • For the application.yml configuration file
  • Marked on the object
  • Assign values to the marked objects
  • The parameter is a SPEL expression

  • First create a properties directory under the resources directory to store custom configurations ending in `.
