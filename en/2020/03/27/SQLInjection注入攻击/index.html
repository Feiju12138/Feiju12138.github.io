【英文】SQL Injection 注入攻击

Posted on 2020-03-27 Edited on 2024-03-28 Views:

Introduction

Handcrafted SQL Injection Attack Study Notes

This article is for learning about defending against network information

Classification by Data Type

  • Numeric SQL injection
    • No need to consider the issue of closing quotes during injection
  • Character SQL injection
    • Need to consider the issue of closing quotes during injection

Classification by Injection Technique

  • UNION query SQL injection (for SELECT operations)
  • Error-based SQL injection (for SELECT operations)
  • Boolean-based blind SQL injection (for SELECT operations)
  • Time-based blind SQL injection (for SELECT operations)
  • Stacked queries SQL injection (for INSERT, DELETE, UPDATE operations)

Classification by Injection Principle (GPC)

  • GET data
  • POST data
  • Cookie data

Data Type Testing

  • Operation: Based on error message, determine the data type of the number stored in the database when 1 is injected
  • Conclusion: When an error message similar to ''' appears, it is judged that the number is stored as numeric data in the database; When an error message similar to '1'' appears, it is judged that the number is stored as character data in the database

Interruption Testing

  • Operation: If paired brackets are interrupted and the following statement is interrupted using comments, it is possible to control the SQL statement
  • Principle: When defining an SQL statement in PHP, brackets are used to wrap the statement, thereby defining the SQL statement as a string

' is used to break the content enclosed by quotation marks

MySQL statement keywords used for interruption: ', ", ), ], }, \, /, ,, .

-- is used to interrupt the following SQL statement

MySQL comment keywords used for interruption: # (needs to be URL-encoded), --

Injection Technique Testing

  • By passing different parameters, judge the page loading and test which injection technique can be used
  • As long as any injection technique can be used, it can be determined that there is an SQL injection vulnerability

Testing for the Existence of Injection Vulnerabilities

  • Operation: Pass different numbers multiple times to determine if the page changes
  • Principle: If id= followed by any number results in dynamically rendering different data on the page, it indicates successful variable parameter passing

Testing Conclusion

  • If the page changes, UNION query injection can be used
  • If the page does not change, continue to determine the result of the Boolean test

Boolean Testing

  • Operation: By passing two statements that return boolean values, judge whether an error occurs on the page
  • Principle: If two statements that return boolean values are passed and the page shows a difference, it means the boolean value return was successful

Testing Conclusion

  • If there is an error, error-based injection can be used
  • If there is no error
    • If the pages are different, boolean blind injection can be used
    • If the pages are the same, continue to determine the result of the Time-based testing

Time-Based Testing

  • Operation: By passing a statement containing the sleep() function, judge whether the page shows a delay (Check via the network option in the developer tools)
  • Principle: The sleep() function can cause a delay effect. If the page successfully loads with a delay, it indicates that the sleep() function was executed successfully

<time>: Specifies the delay time

Testing Conclusion

  • If there is an error, error-based injection can be used
  • If there is no error
    • If the page loads with a delay, time-based injection can be used

UNION Query Injection

  • Principle: The UNION query in MySQL is actually used to find the intersection of two sets and eliminate duplicates. It allows combining a query statement with the previous query statement, but requires that the number of columns obtained by the two SQL statements is the same

  • By specifying the query statement after UNION, it is possible to override the previous query statement and obtain the contents of the fields you want, which will be displayed on the page

  • Necessary condition: Two virtual tables have the same number of columns, and the data types of the columns in the corresponding virtual tables are the same

    • Numeric types can be automatically converted to string types
    • When the necessary conditions are not met, subquery statements can be used

  • Basic Syntax

<select>: Query statement

  • Note: When using Access databases with query statements, the from <database_name> keyword must be used and the database name must exist

Testing Number of Injection Points

  • Principle: The order by statement in MySQL is used to sort data. If a number follows the order by statement, it indicates sorting by the nth field. When there are no fields specified, an error occurs
    • Example: order by 2 means sorting by the second field; if there is no second field, an error occurs
  • Therefore, by using the order by statement, the critical value of the number of fields can be obtained
    • For example: Start testing from order by 1, and when testing order by 3, an error occurs on the page, indicating that the table has only 2 fields and 2 is the critical value
  • Union statement testing can also be directly used to test the number of injection points
    • When select 1,2,... does not cause an error, it signifies the completion of the injection point number testing
  • Change id=1 to another non-existent id to display the content of the select statement after UNION. Alternatively, you can use a statement with a false boolean value (e.g. 1=2) to force the select statement before UNION to return an error, thereby displaying the content of the select statement after UNION

Injection Point Position Testing

  • After determining the number of injection points, you can begin testing the injection point positions
  • Query numbers 1 and 2 (number of digits based on the number of injection points) and check the positions of the injection points on the page
  • Principle: When the value of the select statement before UNION is false, the following select statement is executed

1,2: Content displayed directly on the page (number of digits based on the number of injection points)

Retrieving Data through Injection Points

  • Based on the position echoed by the injection point, change the number of the visible injection point to other functions to retrieve other information

username, password: Field names to be queried, could also be SQL functions, or other subquery statements
user: Table name to query

Enumerating All Table Names in the Database

table_schema: Hexadecimal number of the specified database name

0x: Followed by hexadecimal number, this hexadecimal number is obtained by converting the desired database name to hexadecimal using string to hexadecimal number conversion

concat(): String concatenation function
hex(): Function to convert a string to hexadecimal
database(): Function to get the database name
group_concat(): Concatenates a set of strings into one string, separated by commas

Enumerating All Field Names in a Table

table_schema: Hexadecimal number of the specified database name
table_name: Hexadecimal number of the specified table name

Enumerating Data Content of a Specific Field

users: Table name to be queried
username, password: Field names to be queried
%20: Encoded space string, used to separate multiple pieces of data displayed to prevent confusion
0x3a: Hexadecimal representation of colon string, used to separate multiple pieces of data displayed to prevent confusion
