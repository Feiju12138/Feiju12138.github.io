【英文】XSS 跨站脚本攻击

Posted on 2020-03-25 Edited on 2024-03-28 Views:

Preface

Cross-Site Scripting, or XSS, is a type of code injection that allows malicious users to inject code into a web page, affecting other users who are browsing the page.

This article is only for learning purposes in the field of network information defense.

Conditions

  1. Users are able to control input.
  2. The original program concatenates user input data to execute code.

Types

  • DOM-based XSS
    • XSS is implemented in the request parameters (or may also exist in anchor links), but it only affects the current visit.
  • Reflected XSS
    • The XSS is successfully implemented with the submitted data, but it only affects the current visit and is a non-persistent attack.
  • Stored XSS
    • The XSS is successfully implemented with the submitted data and is stored in the database. It will be automatically triggered when someone visits the page, making it a persistent attack.

Encoding Bypass

  • If the server filters keywords, change the keywords to use alternative character encodings.

Split Bypass

  • If the server filters keywords, split the keyword into multiple string variables and then merge the strings. Execute them using eval().

ShellCode Invocation Locations

  • Write XSS code directly in a JavaScript file and call it remotely.
  • Construct the following request to store the XSS code in the address using window.location.hash.

  • Write the XSS code in the HTML and use AJAX to send a request to invoke it remotely.

  • Write the XSS code in a cookie.

Filter Detection

  • After executing the detection code, inspect the elements to see which characters have been filtered.
  • It can detect whether there is quote filtering, angle bracket filtering, keyword filtering, or case conversion.
