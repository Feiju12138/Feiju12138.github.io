【英文】Feign快速入门

Posted on 2020-11-25 Edited on 2024-03-28 In Java后端学习指北 Views:

Introduction

The functions of Feign include: declarative client, integration with Ribbon (load balancing and retry), integration with Hystrix (fallback and circuit breaker).

Add Dependencies

  • Edit the pom.xml configuration file

Configuration

  • Edit the application.yml configuration file

Add Annotations to the Main Class

  • Add @EnableDiscoveryClient and @EnableFeignClients annotations to the main class

Define the Declarative Client

@FeignClient(): Marks this interface as a declarative client.

name = "": The name attribute defines the spring.application.name name, and the corresponding IP can be found in the Eureka registry through the name.

@GetMapping(), @PostMapping(): Defines the path to access.

@PathVariable: Defines the Rest-style parameter passing (cannot be omitted).
@RequestParam: Defines the ordinary parameter passing (cannot be omitted).

  • If you call the itemClient.method1(1) method, it will send a request to http://localhost:3001/item-service/1.
  • If you call the itemClient.method2(1) method, it will send a request to http://localhost:3001/item-service?id=1.

Load Balancing and Retry

  • No additional configuration is required. Feign has enabled the Ribbon load balancing and retry mechanism by default, and parameters can be adjusted through configuration.

  • Edit the application.yml configuration file (modify all servers)

ConnectTimeout: Connection timeout, default is 1000 milliseconds.
ReadTimeout: Read timeout, default is 1000 milliseconds.
MaxAutoRetries: Number of retries for a single server, default is 0.
MaxAutoRetriesNextServer: Number of retries for multiple servers, default is 1.

  • Edit the application.yml configuration file (modify specific servers)

Fallback and Circuit Breaker

  • Specify the fallback class in the declarative client using the fallback attribute
  • Define the fallback class, implement the declarative client interface, and add fallback code in the overridden methods.
  • The fallback class needs to add the @Component annotation.
  • No configuration is required, it will automatically break the circuit and recover automatically.

Monitoring

  • Feign integrates with Hystrix monitoring.
  • Add the actuator dependency.
  • Expose the hystrix.stream monitoring endpoint.
  • View the monitoring logs.
