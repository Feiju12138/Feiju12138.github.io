Preface

Shiro Framework Learning Notes

Add Dependencies

Shiro Configuration Class

  • Configuration can also be done in a configuration file

  • Create a class as the configuration class for Shiro and annotate it with @Configuration

Shiro Core Configuration Object

  • Create methods in the configuration class to configure Shiro core objects and manage them with Spring

Shiro Access Control Rules Configuration

  • Create methods in the configuration class to configure Shiro access control rules, configure anonymous access resources and authenticated access resources, and manage them with Spring

Default filters in Shiro

anon: Allow anonymous access
authc: Require authentication before access
logout: Logout filter, once executed, the underlying system server will clear the stored user information and directly jump to the login page

**: Represents all files and all subdirectories in the current directory

Create Realm Class

  • Create a Realm class in the service.impl package, inherit AuthorizingRealm class and override abstract methods, annotate this class with @Component to define it as a component managed by Spring

Encrypt the User Input Password

  • Override either the getCredentialsMatcher method or the setCredentialsMatcher method, which is used to return the credential encryption object, and encrypt the user’s input password based on this object
