Introduction

Users and Groups

Users

User Information File

Path

  • /etc/passwd

Meaning of Each Line

  • username:password placeholder:UID:primary group GID:user description:home directory:interpreter

Create New User

<user>: username

Specify Parameters

<option>: option, optional

-u <uid>: specify UID
-g <group>: specify primary group
-G <group>: specify additional groups
-d <home>: specify home directory location
-s <shell>: specify default interpreter

-s /sbin/nologin: specify no interpreter

User Password

Password Information File

Path

  • /etc/shadow

Meaning of Each Line

  • username:encrypted password string:last password change time (days since 1970-01-01):minimum password change time (default 0 means anytime can change):valid days:reminder x days before expiration:::

Change Password

  • When the root user changes any user’s password, password complexity is not restricted, but a prompt will appear, which can be ignored

<option>: option, optional

-l: lock account
-u: unlock account
-d: no password
-f: force user to change password on next login

Change Own Password

  • When other users change their own passwords, password complexity needs to be enforced

Non-Interactive

<pwd>: password

Groups

Group Information File

Path

  • /etc/group

Meaning of Each Line

  • group name:group password placeholder:GID:group member list

Create New Group

<group>: group name
<option>: option, optional

-g <gid>: specify GID
-o: used with -g, allows new user group to have the same GID as existing system user groups

Delete Group

  • Cannot delete a user’s primary group

Modify Group Information

Change Group Name

<name_new>: new group name
<name_old>: old group name

Switch Group

  • If a user belongs to multiple user groups, they can switch between user groups to have the permissions of other user groups
