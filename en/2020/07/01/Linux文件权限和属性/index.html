Introduction

Linux Permissions Study Notes

Viewing Permissions

Viewing the permissions of all files in a specified directory

<dir>: Specify the directory. If not specified, it will display the current directory.

Viewing the permissions of a specific file

<file>: Specify the file.

Viewing the permissions of a directory

Understanding Permissions

Field 1: File Type

-: Regular file
d: Directory
l: Symbolic link
b: Block device

Field 2: Owner’s Read/Write/Execute permissions
Field 3: Group’s Read/Write/Execute permissions
Field 4: Others’ Read/Write/Execute permissions

Modifying File Permissions

<file>: File Path
<attribution>: Attribution

u: Owner
g: Group
o: Others

<authority_type>: Permission Symbol

+: Add permissions
-: Remove permissions
=: Assign permissions

<authority_value>: Permission Value

r: Read permission (value is 4)
w: Write permission (value is 2)
x: Execute permission (value is 1)

Using Numbers to Represent Permission Values

<authority_value>: Permission Value

7: Readable (4), Writable (2), Executable (1)
6: Readable (4), Writable (2), Not Executable (0)
5: Readable (4), Not Writable (0), Executable (1)
4: Readable (4), Not Writable (0), Not Executable (0)
3: Not Readable (0), Writable (2), Executable (1)
2: Not Readable (0), Writable (2), Not Executable (0)
1: Not Readable (0), Not Writable (0), Executable (1)
0: Not Readable (0), Not Writable (0), Not Executable (0)

Numeric Representation of Special Permissions

Sticky Bit
SUID
SGID

Modifying Directory Permissions

-r: Recursive Operation

Modifying File Owner

<user>: Username

Modifying File Group

<group>: Group Name

Sticky Bit

  • Sticky bit is applied to directories. Only the creator of a file in the directory can delete the file.

SUID and SGID

SUID

  • SUID is applied to executable files. Whoever executes the file becomes the owner of the file.

SGID

  • SGID is applied to directories. Files created in the directory inherit the group ownership of the parent directory.

Modifying File Attributes

Viewing all modifiable file attributes

  • View all modifiable file attributes from the help file.

Modifying File Attributes

i: Immutable attribute, files cannot be modified.

Disallow User Creation

  • Modify the attributes of the passwd and shadow files by adding the immutable attribute to disallow user creation.

Default Permissions

  • Default file permissions = Default highest permission for files (666) - Current user’s umask value
  • Default directory permissions = Default highest permission for directories (777) - Current user’s umask value
    • Default umask value for regular users: 0002
    • Default umask value for root user: 0022
    • Recommended default umask value in enterprises: 0027
