Preface

MongoDB is a document-oriented database management system written in languages such as C++, designed to solve a plethora of real-world problems in the application development community. MongoDB was developed by MongoDB Inc. (then the 10gen team) in October 2007 and released for the first time in February 2009, distributed under the Server Side Public License (SSPL). (Wikipedia)

Differences Between MongoDB and Other Databases

  • Databases are represented by database
  • Collections (equivalent to tables in other databases) are represented by collection
  • Fields (equivalent to columns in other databases) are represented by field
  • MongoDB commands are case-sensitive
  • MongoDB commands do not require a semicolon at the end.

Data Types

  • MongoDB stores data using BSON (a binary version of JSON)
  • BSON Data Types
    • Data types that can be used in the Shell
      • String
      • Object ID
      • Boolean
      • Array
      • 32-bit integer
      • 64-bit integer
      • 64-bit floating point number
      • null
      • undefined
      • Regular expression //i
      • function(){}
    • Data types that cannot be used in the Shell
      • Symbol
      • Binary data
      • Max/Min values

Operations on Databases

Switching (Creating/Using) Databases

  • After executing the use command, you will enter the database with the specified database name, regardless of whether the database name exists or not.
    • If the database name already exists, you will enter an existing database for operations.
    • If the database name does not exist, the database will be created automatically after creating a collection. It will still not appear in the list of all databases until a collection is created.

<database_name>: Database Name

Database Naming Rules

  • Cannot be an empty string
  • Cannot contain null character (\0)
  • Cannot contain spaces ( )
  • Cannot contain special characters: ., \, /, $
  • Must be in all lowercase letters
  • Maximum length of 64 bytes

Default Databases

  • admin: Stores all users and permissions
  • local: Data stored in this database will not be synchronized after deploying a cluster
  • config: Stores sharding information
