Preface

Java Collection (java.util.Collection) study notes
Collection is used to store a group of elements. It provides related operations for maintaining collections.

Whether the elements are duplicated depends on the result of comparing the elements’ equals method.

Inheritance structure of Collections

graph TD
A[Collection interface] --> B[List interface]
    A --> C[Set interface]
    A --> D[Map interface]
    B --> E[ArrayList subclass]
    B --> F[LinkedList subclass]
    C --> G[HashSet subclass]
    D --> H[HashMap subclass]

List

  • It is a repeatable and ordered collection. The feature is that elements can be operated based on their index.

  • Common implementation classes:

ArrayList: It uses an array as the underlying implementation, which makes querying faster.
LinkedList: It uses a linked list as the underlying implementation, which makes insertion and deletion faster (the effect of insertion and deletion at the beginning and end is obvious).

Set

  • A collection without duplicates

Map

  • Also called a hash table or hash map
  • Commonly used for storing data in key-value pairs
  • The keys cannot be duplicated, but the values can be duplicated

Generics

  • Generics, also known as parameterized types, give the definition of the type of the current class’s properties, method parameters, and method return values to the user. Users can pass the actual type of the generics while creating the current class.

  • The prototype of generics is Object, and the definition of generics is only for the compiler to do some validation work. When we set a generic type, it will check whether it meets the type requirements. When we get a value of a generic type, it will automatically perform type conversion.

Sorting

  • The sorting collection uses the static method sort of the collection’s utility class Collections

  • Sorting can only be performed on List collections

Custom sorting rules

  • After implementing the Comparable interface, you need to rewrite the following method. The purpose of this method is to define the rules for comparing the size between the current object and the given parameter object.
  • The return value is an int value, which represents the size relationship. It does not care about the specific value, but focuses on the value range.
  • When the return value > 0: The current object is larger than the parameter object
  • When the return <0: The current object is smaller than the parameter object
  • When the return value = 0: The two objects are equal.

Overloaded methods

  • The sort method requires that the collection elements must implement the Comparable interface, which specifies that the implementation class can be compared. One of its abstract methods is used to define the size rules.
  • We want to use the sort method to sort the collection, but this method requires that our collection elements must implement the Comparable interface and define comparison rules. This phenomenon that we want to use a certain function, but it requires us to modify the program is called “invasiveness”. The more code we modify, the greater the invasiveness, and the less conducive to program expansion.
  • The overloaded sort method requires passing an additional comparator, which no longer requires the collection elements to implement the Comparable interface, nor does it use the collection elements’ own comparison rules to sort the class, but sorts the collection elements based on the comparison rules of the given additional comparator.
  • In actual development, it is also recommended to use this method to sort collection elements if the collection elements are custom. When creating a comparator, it is also recommended to use the form of anonymous inner class.
