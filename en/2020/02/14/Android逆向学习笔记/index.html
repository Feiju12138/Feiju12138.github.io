Preface

Notes on Android reverse engineering

APK File Structure

assets

  • Uncompiled resource files

lib

  • Dynamic link libraries, which store .so files

META-INF

  • File digest, encrypted digest, and signing certificate file directory

CERT.RSA

  • Public key and encryption algorithm description

CERT.SF

  • Encrypted file, which is the encrypted ciphertext obtained by encrypting the plaintext digest with the private key. Only the public key paired with the private key can decrypt this file.

MANIFEST.MF

  • Manifest file, which contains the plaintext digest of all files in the package

res

  • Resource file directory in binary format

drawable

  • Image resources

layout

  • Layout resources

menu

  • Menu resources

resources.arsc

  • Compiled resource files

classes.dex

  • Executable file

AndroidManifest.xml

  • Configuration file

APK Packaging Process

  • Package resource files and generate R.java and resources.ap_ files
  • Process AIDL files and generate corresponding .java files
  • Compile Java files and generate corresponding .class files
  • Convert .class files into .dex files supported by Dalvik VM
  • Package and generate unsigned .apk file
  • Sign the unsigned .apk file
  • Align the signed .apk file

APK Installation Process

Installation Methods

System Programs

  • Installation of system programs

User Programs

  • Manual installation of installation package

  • Installation via application market

  • Installation via ADB

Installation Process

  • Copy the APK installation package to the /data/app directory, extract and scan the installation package
  • Save the dex file to the /data/dalvik-cache directory
  • Create the corresponding application data directory under the /data/data directory

Directory Structure

/system/app

  • System-installed applications that require root permission to delete

/data/app

  • Directory where user-installed programs are installed. APK files are copied to this directory during installation.

/data/data

  • Store application data

/data/dalvik-cache

  • Install dex in the dalvik-cache directory

Uninstallation Process

  • Delete the files and directories created in the above three directories during installation

APK Decompilation and Recompilation

  • Check the shell first, then decompile for verification
  • Convert .dex files to .smali files using the decompilation tool apktool
  • Remove ads and pop-ups. If the Activity cannot be found in smali, analyze the code and program logic, capture key information, interfaces, and functions.

Decompilation on MacOS

Preparation

  • Apktool
  • dex2jar
  • JD-GUI

Decompiling APK Files

  • Use the apktool command to decompile the .apk file

<name>: .apk file

Decompiling DEX Files

  • Extract the .apk file directly, or use the apktool command to recompile the recently decompiled directory

<directory>: directory being decompiled

  • Use the d2j-dex2jar command to decompile the .dex file
  • Open the .jar file using the JD-GUI software
