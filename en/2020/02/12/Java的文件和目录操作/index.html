Preface

Java implements the operation of files and directories as well as file reading and writing through IO streams.

File Object

  • Encapsulates a disk path string.

String

File path
Folder path
Nonexistent path

  • Provides some operations on files or directories.

Creating Objects

  • Whether the file exists or not, a new empty file will be created.
  • If the directory does not exist, an exception will be thrown.

<directory>: directory name
<file>: file name

Directory listing

IO Stream

  • Abstract data flowing in the pipeline.
  • One-way.

Input stream, can only read data.
Output stream, can only output data.

  • Data can only flow through from start to finish once.

Byte Stream

Byte Input Stream

InputStream
  • Parent class is an abstract class, cannot create objects.
  • Byte streams can handle files of any data.
FileInputStream
  • The subclass inherits from the parent class, can create objects.
  • Reads one byte at a time.
Creating objects

<str>: String type file path

<file>: File object encapsulating the file path

BufferedInputStream
  • The subclass inherits from the parent class, can create objects.
  • Reads one array at a time.
  • The default length of the buffer array is 8192.
Creating objects
  • Any InputStream subclass can be passed as an object parameter.
