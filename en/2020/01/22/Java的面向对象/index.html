【英文】Java的面向对象

Posted on 2020-01-22 Edited on 2024-03-28 In Java后端学习指北 Views:

Preface

Java Object-Oriented Programming (OOP) Study Notes

Classes and Objects

  • The real world is made up of many objects
  • Classes are extracted based on objects

Concepts

  • Object: A real and individual entity
  • Class: Type/category, representing a category of entities

A class can include

  • Member variables

Properties/attributes shared by all objects

  • Methods

Behaviors shared by all objects

Characteristics

  • A class can create multiple objects
  • Objects of the same type have the same structure but different data
  • A class is a template for objects, and objects are specific instances of classes

Methods

Method Signature

  • Method name + parameter list

Method Overloading

  • Occurs within a class where method names are the same, but the parameter lists and method bodies are different
  • The compiler automatically binds the called method based on the method signature during compilation

Different numbers of parameters with the same parameter type

Constructors

  • Constructor, builder

  • Initialize member variables

  • Have the same name as the class and no return type

  • Automatically called when an object is created

  • If no constructor is written, the compiler will provide a default parameterless constructor

  • If a constructor is written, the default constructor will no longer be provided

  • Constructors can be overloaded

this

  • Refers to the current object, it refers to the object that calls the method

  • Can only be used in methods, there is a default “this.” before accessing member variables in the method

Uses of this

Accessing member variables
Calling a method (generally not used)
Calling a constructor

null

  • null, does not point to any object

  • If a reference has a value of null, the reference cannot perform any further operations

  • If an operation is performed, a NullPointerException will occur

Reference Type Variables

  • Point to the same object
  • Modifying data through one reference affects accessing data through another reference

Primitive Type Variables

  • Assign a value
  • Modifying one variable does not affect another variable

Reference Type Arrays

Inheritance

Purpose

  • Code reuse

Implementation

  • Implemented using extends

Scope of Attributes and Behaviors

  • Superclass/Parent class: Attributes and behaviors shared by all subclasses
  • Subclass/Child class: Attributes and behaviors specific to the subclass
  • After the subclass inherits from the superclass, the subclass has: subclass’s + superclass’s

Characteristics

  • A superclass can have multiple subclasses

  • A subclass can only have one superclass (single inheritance)

  • Transitive

Rules

  • The superclass must be constructed before the subclass can be constructed

  • If the superclass’s constructor is not called in the subclass’s constructor, the default super() will be called

  • If the superclass’s constructor is called in the subclass’s constructor, the default constructor will not be provided

super()

  • Calls the superclass’s constructor and must be the first line in the subclass’s constructor

super

  • Refers to the superclass object of the current object

Uses of super

Accessing superclass member variables

Accessing superclass member variables
Calling a superclass method
Calling a superclass constructor

Upcasting

  • The reference of a superclass points to an object of a subclass
  • The capabilities of the reference depend on its type

Method Overriding

  • Rewriting, overriding

  • Occurs in parent and child classes where method names are the same, the parameter lists are the same, but the method bodies are different

  • When the overridden method is called, it depends on the type of the object

Follow the principle

  • Follow the “two same, two small, one big” principle

two same

  • Same method name
  • Same parameter list

two small

  • The return type of the subclass method is smaller than or equal to the return type of the superclass method

If it is void, it must be the same
If it is a primitive type, it must be the same
If it is a reference type, it must be smaller than or equal to

  • The exceptions thrown by the subclass method are smaller than or equal to the exceptions thrown by the superclass method

one big

  • The access modifier of the subclass method is greater than or equal to the access modifier of the superclass method

package

Purpose

  • Avoid naming conflicts

Characteristics

  • Package names can have a hierarchical structure, and classes in the same package cannot have the same name

Full name of a class

Recommendation

  • All lowercase letters for package names

import

  • Classes in the same package can be accessed directly
  • Classes in different packages cannot be accessed directly, there are two ways to access them

Import the class first and then use the class
Full name of the class

Access Control Modifiers

public: Public, any class
protected: Protected, the current class, subclasses, and classes in the same package
Default: No modifier, the current class and classes in the same package
private: Private, the current class

  • The access modifier of a class can only be public or the default
  • The access modifiers of members in a class can be any of the above four types

final

  • Final, cannot be changed

Modifying Variables

  • Variables cannot be changed

Modifying Methods

  • Methods cannot be overridden

Modifying Classes

  • Classes cannot be inherited

static

  • Static

Static Variables

  • Modified
