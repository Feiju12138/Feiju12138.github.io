Container operations

Run a container

<name>:<version>: Specify the image by name and tag
<image_id>: Specify the image by unique identifier

Parameters

-d: Run the container in the background
-p: Map the port of the current host to the port of the container

<port_out>: Host port
<port_in>: Container port
--name <name>: Specify the container name
<name>: Container name
-e: Add environment variables in the container
-i: Enable interaction, usually used with -t
-t: Provide a terminal
--rm: Automatically delete the container when it is stopped
--restart=always: Automatically start the container, the container will also start automatically when Docker is restarted
