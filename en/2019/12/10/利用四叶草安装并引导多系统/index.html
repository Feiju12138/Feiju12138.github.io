【英文】利用四叶草安装并引导多系统

Posted on 2019-12-10 Edited on 2024-03-28 Views:

Preface

After untiring efforts, I finally managed to install multiple operating systems on the same hard drive and successfully boot them using Clover.

Budget before installation

  • I used a Dell desktop computer in the machine room, with a hard disk size of 1T, so I decided to perform installation tests for 4 systems, including disk allocation and installation order, as shown in the table below:

Since it is only for experimental purposes and not for office use, the disk allocation size is slightly smaller.

Installation Order System Disk Allocation Size
1 MacOS 128G
2 Windows8 128G
3 Ubuntu 64G
4 CentOS 64G

Preparation

  • USB drive with capacity of at least 8G for burning various system installation disks (preferably not 8G)

  • Prepare a USB drive with a system maintenance disk (WindowsPE) for Big Cabbage

  • Additional computer that can connect to the Internet for reference (if there are any pitfalls not mentioned by me, please search them on Baidu yourself)

First partition

  • Insert the system maintenance USB drive, enter the WindowsPE system, and use the “DiskGenius” tool to delete all partitions on the disk

  • Create the ESP partition with a size of 512M

Smaller partitions can actually be created, but I just want to make sure it’s enough.

  • Create an NTFS partition with a size of 128G for installing Hackintosh, and then save and format it

Install Hackintosh

  • First of all, I installed Hackintosh because the Hackintosh installation package I used is a “lazy” package (system image + Clover bootloader), and the Clover bootloader is already included in the installation package. Therefore, it is preferred to install Hackintosh to facilitate booting other systems.

  • When partitioning in the previous step, a 128G NTFS formatted hard disk should be partitioned, and then enter the MacOS utility tool and format it as MacOS (journalized) format. Do not directly partition 128G MacOS (journalized) using DiskGenius.

Link

  • The first thing to do after installing is to add the Clover bootloader to the EFI partition of the system.

Link

Second partition

  • Create a new partition with a size of 128G, format it as NTFS, and save and format it

Install Windows

  • The second installation is the Windows operating system. The first time I installed it, I installed Windows10. However, during the process of researching other systems, the bootloader was automatically replaced. Therefore, the second time I installed Windows8 during the experiment.

It doesn’t mean that Windows10 cannot be successfully installed, it’s just that I want to try as many new system installations as possible and document them.

Link

  • After installing it, in order to facilitate disk partitioning, I downloaded the Windows UEFI Edition of Big Cabbage and installed the WindowsPE system on the computer

In fact, after installing “Big Cabbage” into the system, it is not an independent partition that can be booted through the bootloader. Instead, it is integrated into the Windows system. Therefore, after booting into Windows, select “Big Cabbage” to enter the PE system.

  • At this time, the Clover bootloader does not need any modifications. It can still automatically recognize Hackintosh and Windows. For safety reasons, use the “DiskGenius” tool to enter the EFI partition, click “Browse Files”, enter the EFI folder, and you can see the following folders. Save them to a USB drive to prevent overwriting.

APPLE: Bootloader for MacOS
BOOT: Regular UEFI bootloader
CLOVER: Clover bootloader
Microsoft: Bootloader for Windows

Third partition

  • Create a new partition with a size of 64G and format it as ext4 (or format it as NTFS and reformat it when installing Ubuntu)

Install Ubuntu

Link

  • In the “Installation Type” stage, select “Something else” option. Set at least one mount point as the root directory “/“, formatted as ext4.

  • After successfully installing Ubuntu, there may be an issue with invalid booting. Enter the PE system and use the “DiskGenius” tool.

  • Insert the USB drive for storing files (can be the Big Cabbage USB drive).

  • Find the Clover configuration file config.plist

/EFI partition/EFI folder/CLOVER/config.plist

  • Copy config.plist and paste it into another computer that can be used normally

  • Edit config.plist, find the keyword GUI, and modify the content as shown below

  • Copy the <dict></dict> tag and modify the content of the string<></string> tag to the EFI parameter for Ubuntu

\EFI\ubuntu\grubx64.efi

  • Overwrite the system’s original file with the modified file

Fourth partition

  • Create a new partition with a size of 64G and format it as ext4

Install CentOS

Link

  • Similar to Ubuntu, in the “Installation Destination” step, select “Customize storage layout” option, set at least one mount point for the root directory “/“ and set the location for “efi” as “/boot/efi”

  • The problem encountered is still the possibility of EFI being overwritten. Remember to back up all contents of the EFI partition before installation, and modify the Clover configuration file after installation

Completion

  • This multiple system installation journey comes to an end, and all systems have been tested and are functioning properly. The above are the most basic 4 types of operating systems (in other words, 3 types of operating systems, two of which are Linux systems, but they belong to different categories and still have differences). Installing Red Hat, a few of the operations are the same, so you can continue to install them.

  • Finally, the successful case screenshot is attached (using the F10 screenshot function of Clover created by “Black Apple Soldier”), hoping that everyone can successfully explore.

Postscript

  • Continue using the Clover bootloader to install more systems
