Preface

Android Debug Bridge (ADB) is a versatile command-line tool that allows you to communicate with devices. ADB commands facilitate various device operations (such as installing and debugging applications, transferring files, etc.) and provide access to the Unix shell (which can be used to run various commands on the device). It is a client-server program, consisting of the client, daemon (adbd), and server components. (Wikipedia)

Device Connection Management

USB Connection

  • Hardware device functioning properly

  • Android Developer Mode and USB Debugging Mode are enabled

  • Device driver functioning properly

  • USB connection working

WLAN Connection (requires USB cable)

  • Device and computer are on the same wifi network

  • USB connection between device and computer is working

  • Computer listening on port 5555

  • Disconnect USB connection

  • Identify device IP address

  • Connect to device using IP address

WLAN Connection (without requiring USB cable)

  • Device and computer are on the same wifi network

  • Open a terminal on the device (using the system’s built-in terminal or third-party software like Termux)

  • Open port

  • Connect to device using IP address

Log Printing

Filter logs by level

<priority>: Priority

*:V: Show only Verbose and higher levels (lowest priority)
*:D: Show only Debug and higher levels
*:I: Show only Info and higher levels
*:W: Show only Warning and higher levels
*:E: Show only Error and higher levels
*:F: Show only Fatal and higher levels
*:S: Show only Silent and higher levels (highest priority)
