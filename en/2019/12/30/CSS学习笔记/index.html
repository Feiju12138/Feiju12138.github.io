    Introduction

    Cascading Style Sheets is a computer language used to add style to structured documents, defined and maintained by W3C. The current latest version is CSS2.1, which is a W3C recommended standard. CSS3 is now supported by most modern browsers, while the next version CSS4 is still in development. (Wikipedia)

    Syntax Rules

    Usage

    Inline Style

    • Written inside HTML tags

    style declaration: style property: style value

    Internal Style

    • Write styles inside <style></style> tags
    • <style></style> tags are placed inside <head></head> tags

    External Style

    • Reference external .css file inside <head></head> tags

    href: resource path

    Specificity

    • When multiple styles are applied to an element and there is a conflict, the styles are applied based on specificity rules

    Lowest: Browser default styles
    Closest Priority: Whichever style is closest to the element is chosen
    Highest: Inline styles

    Specificity Values of Selectors

    • Indicates the importance of the selector, higher the value, higher the priority
    • When a selector contains multiple selectors, the values are added and the selector with higher value is applied
    • Same values, closest rule is followed
    • !important directly gets the highest priority, inline styles are not allowed to use !important
    • Group selector values are calculated individually and cannot be combined
    • The calculation of selector values cannot exceed its maximum significant figure

    !important > 1000
    Inline styles = 1000
    ID selectors = 100
    Class/Pseudo-class selectors = 10
    Element selectors = 1
    Universal selectors = 0
    Inherited styles = null

    Adjusting Specificity

    • Place it after the property, separated by a space from the property value, to adjust the specificity of that property to the highest level

    Content Generation

    • Adds HTML elements using CSS commands
    1. Pseudo-elements can only be added to paired tags
    2. Must have a content attribute
    3. Must be out of the standard document flow

    • The content attribute in pseudo-elements sets the text content, by default can pass an empty string

    • Only two pseudo-elements can exist within the same tag, if more than two, the latter overrides the former

    Sizes

    • Set the width and height of elements

    Elements Allowed to Have Set Sizes on a Page

    span: for setting background color width and height
    div: for setting background color, width, and height
    input: for setting width and height

    • Block-level elements can have width and height set, if width is not set, default width is 100% of the parent element, if height is not set, default height depends on content
    • Inline elements’ width and height settings are ineffective, width and height are determined by content
    • Inline-blocks can have width and height set, inline-blocks have default width and height, different browsers interpret this differently (span, input)
    • Elements with default width and height attributes can have styles applied for width and height (img, table)

    Units

    Absolute Units

    px: pixels, logical pixels and physical pixels
    in: inches, 1in = 2.54cm
    pt: points, 1pt = 1/72in, commonly used for font sizes
    cm: centimeters
    mm: millimeters

    Relative Units

    %: percentage of parent element’s width
    em: relative to the font size of the element
